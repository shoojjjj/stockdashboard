# 텔레그램 원천파서 운용

> 역할: Telegram 원천을 날짜, 본문, 사진, PDF/파일 단위로 구조화해 후속 OCR/태그화용 인덱스를 만든다.
> 기본 경로는 Telegram API 수집이고, Desktop HTML Export는 백업/검증 경로로만 둔다.

## 기본 경로: Telegram API

API 수집 운용은 아래 문서를 따른다.

- [텔레그램 API 수집 운용](<텔레그램_API_수집_운용.md>)
- 수집기: `telegram_api_collect.py`
- 기본 출력: `API_수집/인덱스/소수몽키_API_원천인덱스.jsonl`
- OCR 원천 루트: `01_텔레그램_원천파서/API_수집`

```bash
python3 0_주식_에이전트/소수몽키_에이전트/01_텔레그램_원천파서/telegram_api_collect.py \
  --from-date 2026-05-28 \
  --to-date 2026-05-30 \
  --limit 1000 \
  --download-media
```

API 인증값은 환경변수나 `.env`로 제공한다. 세션 파일과 인증값은 문서/브리핑 폴더에 복사하지 않는다.

## 보조 경로: Desktop Export

## 입력

```text
소수몽키/Telegram Desktop/ChatExport_2026-05-30/messages.html
소수몽키/Telegram Desktop/ChatExport_2026-05-30/messages2.html
소수몽키/Telegram Desktop/ChatExport_2026-05-30/messages3.html
소수몽키/Telegram Desktop/ChatExport_2026-05-30/messages4.html
```

## 추출 규칙

| 필드 | 추출 기준 |
|------|-----------|
| 메시지 ID | `div.message`의 `id` |
| 게시 일시 | `div.date.details`의 `title` |
| 본문 | `div.text` 내부 텍스트, 링크 텍스트 포함 |
| 사진 | `a[href^="photos/"]` |
| PDF/파일 | `a[href^="files/"]` |
| 서비스 메시지 | `div.message.service`로 분리 집계 |

## 사진 처리

- 링크의 `href`가 가리키는 원본 사진만 인덱싱한다.
- 이미지 태그의 `src`가 `_thumb` 파일을 가리키더라도 OCR 대상에는 넣지 않는다.
- 같은 분 단위로 이어진 사진은 후속 단계에서 하나의 카드 묶음 후보로 본다.

## PDF 처리

- `files/` 링크는 파일명, 게시 시각, 메시지 ID만 기록한다.
- PDF 본문 추출은 1차 범위에서 제외한다.

## 실행

Desktop Export를 재검증하거나 API 장애 시 아래 경로를 사용한다.

```bash
python3 0_주식_에이전트/소수몽키_에이전트/01_텔레그램_원천파서/telegram_export_index.py \
  --export-dir "소수몽키/Telegram Desktop/ChatExport_2026-05-30" \
  --output-dir "0_주식_에이전트/소수몽키_에이전트/01_텔레그램_원천파서/인덱스" \
  --from-date 2026-05-28 \
  --to-date 2026-05-30
```

## 출력

| 파일 | 설명 |
|------|------|
| `API_수집/인덱스/소수몽키_API_원천인덱스.csv` | API 메시지 단위 인덱스 |
| `API_수집/인덱스/소수몽키_API_원천인덱스.jsonl` | API 후속 OCR/태그화용 구조화 인덱스 |
| `API_수집/인덱스/소수몽키_API_일자별_집계.csv` | API 날짜별 메시지, 텍스트, 사진, 파일 수 |
| `API_수집/인덱스/소수몽키_API_최근3일_인덱스.md` | API 최근 구간 검토용 인덱스 |
| `인덱스/소수몽키_원천인덱스_전체.*` | Desktop Export 보조 인덱스 |

## 검증

- API 수집 결과의 날짜 범위와 메시지 수가 요청 구간과 맞는지 확인한다.
- `--download-media`를 사용한 경우 `API_수집/photos/` 파일이 OCR 샘플러에서 열려야 한다.
- Desktop Export 병행 검증 시 전체 날짜 범위가 `2026-04-01 ~ 2026-05-30`인지 확인한다.
- Export 경로의 `photos/*_thumb.jpg`는 OCR 후보에 포함하지 않는다.
