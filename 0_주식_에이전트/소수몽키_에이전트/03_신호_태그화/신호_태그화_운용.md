# 신호 태그화 운용

> 역할: 텍스트 메시지와 OCR 샘플을 합쳐 종목, ETF, 섹터, 매크로, 이벤트 신호를 태그화한다.

## 입력

- 원천 인덱스 JSONL
- OCR 샘플 JSONL

## 태그

| 태그 | 예시 |
|------|------|
| 종목 | 개별 기업명, 티커 |
| ETF | QQQ, SPY, IWM, SMH, IBIT 등 |
| 섹터 | AI, 반도체, 전력, 원전, 방산, 에너지, 금융 |
| 매크로 | 금리, 환율, 유가, 인플레이션, 경기 |
| 이벤트 | 실적, 가이던스, 정책, 관세, 회담, 보고서 |
| 실적 | 매출, EPS, 마진, CAPEX |
| 정책 | 세금, 규제, 보조금, 정부 지출 |
| 리스크 | 과열, 조정, 관세, 유동성, 지정학 |

## 신호 등급

| 등급 | 의미 |
|------|------|
| `SM-S0` | 단순 소음 또는 반복성 낮음 |
| `SM-S1` | 배경 정보, 일일 요약 후보 |
| `SM-S2` | 반복 등장하거나 시장 영향이 있는 감시 신호 |
| `SM-S3` | 소수몽키 관점의 핵심 주장 또는 강한 변화 |

## 판정 원칙

- 단발 언급과 반복 언급을 분리한다.
- 사진, 텍스트, PDF 링크가 같은 방향을 가리키면 등급을 올릴 수 있다.
- `SM-S2`, `SM-S3`는 과도하게 많이 부여하지 않는다.
- 매수, 매도 결론 대신 관찰 신호와 다음 확인만 남긴다.

## 산출 형식

```text
날짜:
자료 유형:
핵심 테마:
언급 종목/ETF:
이벤트:
강도:
근거 위치:
소수몽키 관점:
주의점:
다음 확인:
```

## 출력 위치

```text
0_주식_에이전트/소수몽키_에이전트/03_신호_태그화/신호판/
```
