# 소수몽키 총괄 운용

> 작성일: 2026-05-30
> 범위: Telegram API를 기본 원천으로 읽어 원천 인덱스, OCR 샘플, 신호판, 일일브리핑을 만든다.
> 원천: Telegram API 수집. 기존 `소수몽키/Telegram Desktop/ChatExport_2026-05-30`는 보조/검증 원천으로 유지한다.

## 호출 조건

- 사용자가 소수몽키 Telegram API 수집, 소수의견 채널, 소수몽키 관점 브리핑을 요청할 때 호출한다.
- 자산제곱 에이전트와 교차 판단하지 않는다. 1차 산출물은 소수몽키 독립 관찰로 둔다.
- 매수, 매도, 비중 결론을 직접 내리지 않는다. 신호, 반복성, 리스크, 다음 확인만 기록한다.
- `오늘 매매할 것은?` 요청에서는 소수몽키 일일브리핑과 신호판을 [토론 중재 에이전트](../../토론_중재_에이전트/README.md)에 넘긴다.

## 전체 흐름

```text
Telegram API 수집
-> 01_텔레그램_원천파서
-> 02_이미지_OCR_판독
-> 03_신호_태그화
-> 04_일일브리핑
```

## 산출물 위치

| 단계 | 위치 | 설명 |
|------|------|------|
| API 원천 인덱스 | `01_텔레그램_원천파서/API_수집/인덱스/` | API 메시지, 사진, 파일을 날짜별로 재구성한다. |
| Export 보조 인덱스 | `01_텔레그램_원천파서/인덱스/` | Desktop Export 검증용 인덱스다. |
| OCR 샘플 | `02_이미지_OCR_판독/OCR_샘플/` | 최근 3일 원본 사진 일부를 OCR한다. |
| 신호판 | `03_신호_태그화/신호판/` | 종목, ETF, 섹터, 매크로, 이벤트를 `SM-S0~SM-S3`로 태그화한다. |
| 일일보고 | `1_브리핑/소수몽키_브리핑/일일보고/` | 소수몽키 관점의 오늘 핵심과 반복 테마를 정리한다. |

## 운영 원칙

- Telegram API 수집을 기본 경로로 둔다.
- API ID/hash, `.env`, Telethon 세션 파일은 문서와 브리핑 산출물에 넣지 않는다.
- 원천 Export는 이동하거나 수정하지 않고 API 장애 시 보조 검증에만 쓴다.
- 썸네일 파일명에 `_thumb`가 들어간 이미지는 OCR 대상에서 제외한다.
- PDF는 1차 범위에서 전문 판독하지 않고 파일 링크와 게시 시각만 인덱싱한다.
- OCR은 기본적으로 로컬 `tesseract kor+eng --psm 6`을 사용한다.
- 외부 비전 API나 자산제곱 연결은 별도 요청이 있을 때만 확장한다.

## 토론 연결

소수몽키 에이전트는 토론에서 매수/매도 버튼을 누르지 않는다. 아래 세 가지만 말한다.

```text
강한 신호:
반복 테마:
리스크/과열:
```

중재자는 이 신호를 자산제곱 게이트와 포트폴리오 여력에 대조한다.

## 1차 완료 기준

- API 수집 기준 최근 3일(`2026-05-28`, `2026-05-29`, `2026-05-30`) 메시지, 사진, 파일 수가 집계된다.
- 보조 Export 기준 `2026-04-01 ~ 2026-05-30` 전체 날짜 범위가 확인된다.
- OCR 샘플에서 티커, 종목명, ETF, 수익률, 핵심 키워드 일부가 추출된다.
- `2026-05-30` 기준 신호판과 일일브리핑 샘플이 작성된다.
