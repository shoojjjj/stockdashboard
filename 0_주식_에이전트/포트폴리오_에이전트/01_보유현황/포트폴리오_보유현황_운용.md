# 포트폴리오 보유현황 운용

> 역할: 사용자 제공 계좌 스냅샷을 실제 보유현황 문서로 정리한다.

## 입력

- 계좌별 보유 종목, 수량, 평가금, 손익률
- 원화/달러 예수금
- 계좌별 성격: 장기/코어, 스윙/전술, 현금/방어

## 출력 위치

```text
1_브리핑/포트폴리오_브리핑/00_현황/실제_보유현황.md
```

## 기록 원칙

- 사용자 제공 숫자는 `계좌 데이터`로 표시한다.
- `G3`, `R1`, `수익 보호`, `추가 금지` 같은 판정명은 에이전트 운용규칙으로 표시한다.
- 보유 스냅샷은 원본 계좌 데이터를 대체하지 않는다.
- 숫자가 오래되었으면 날짜를 유지하고 최신처럼 쓰지 않는다.
- 실행 후 새 날짜 문서를 만들지 않는다. 최신 보유 상태는 `실제_보유현황.md`에 덮어쓴다.
- 목표 포트폴리오와 리밸런싱 계획 변경은 `1_브리핑/포트폴리오_브리핑/01_목표구성/목표_및_리밸런싱_현재판.md`에 반영한다.

## 점검 항목

| 항목 | 질문 |
|------|------|
| 보유 이유 | 아직 원래 역할이 살아 있는가? |
| 계좌 역할 | 장기/코어와 스윙/전술이 섞이지 않았는가? |
| 손익 상태 | 수익 보호 또는 정리 검토가 필요한가? |
| 현금 여력 | 조건부 정찰병을 감당할 수 있는가? |
| 중복 노출 | 같은 테마가 과도하게 쌓였는가? |
