# 자산제곱 시나리오 사고

> 목적: 자산제곱식 사고에서 `시나리오`가 무엇인지, 어떻게 만들고 검증해야 하는지 정리한다.
> 작성일: 2026-05-23
> 상위/입력 문서: [자산제곱 총괄 브레인](<../00_총괄브레인/자산제곱_총괄브레인.md>), [자산제곱 칼럼 인입/반영/분석](<../01_칼럼인입/자산제곱_칼럼인입_반영_분석.md>), [자산제곱 이벤트/시나리오 업데이트](<../02_이벤트시나리오/자산제곱_이벤트시나리오.md>), [자산제곱 철학](<../03_철학/자산제곱_철학.md>)
> 범위: 이 문서는 시장 전망, 종목 추천, 이번 주 매매 판단을 쓰는 문서가 아니다.
> 금지: 특정 종목의 매수 가격, 보유 수량, 오늘의 매매 결론을 적지 않는다.

## 한 줄 정의

자산제곱식 시나리오 사고는 `미래를 하나로 맞히는 예측`이 아니라, 가능한 미래를 여러 개로 나누고 각각의 확률, 트리거, 대응, 무효화 기준을 미리 정하는 판단 체계다.

출처 구분:

- 칼럼이 직접 제시한 이벤트, 시장 경로, 수치, 종목은 `칼럼 원문`으로 다룬다.
- 시나리오 개수, 확률 범위, 트리거 표, 무효화 양식은 `에이전트 운용규칙`이다.
- 에이전트가 부여한 확률과 대응 문장을 자산제곱의 직접 전망처럼 표현하지 않는다.

## 철학과의 관계

철학 문서가 `왜 그렇게 생각해야 하는가`를 다룬다면, 시나리오 문서는 `불확실한 미래를 어떤 구조로 나누어 볼 것인가`를 다룬다.

관계:

- 철학은 상위 기준이다.
- 칼럼 인입은 새 정보가 시나리오로 들어올지 선별하는 입구다.
- 이벤트/시나리오 업데이트는 날짜가 있는 변수와 종료 조건을 먼저 관리한다.
- 시나리오는 철학을 복수의 미래로 번역하는 장기 사고 도구다.
- 판단원칙은 시나리오를 실제 운용 기준으로 바꾸는 다음 단계다.
- [매수게이트](<../07_매수게이트/자산제곱_매수게이트.md>)는 시나리오 중 실행 가능한 후보만 걸러내는 하위 단계다.

## 시나리오 에이전트의 역할

시나리오 에이전트는 아래 질문에 답하는 역할을 한다.

1. 지금 논점은 하나의 결론으로 단정할 수 있는가, 복수 시나리오로 봐야 하는가?
2. 시장을 움직이는 핵심 변수는 무엇인가?
3. 가능한 시나리오는 몇 개인가?
4. 각 시나리오의 확률은 어떤 근거로 배정되는가?
5. 어떤 신호가 나오면 특정 시나리오가 강화되는가?
6. 어떤 신호가 나오면 기존 시나리오가 무효화되는가?
7. 각 시나리오에서 포트폴리오와 비중은 어떻게 달라져야 하는가?

## 시나리오로 다룰 대상

| 대상 | 시나리오 적용 방식 |
|------|-------------------|
| 시장 국면 | 상승 지속, 조정, 급락, 순환매, 유동성 장세를 나눠 본다. |
| 정책 이벤트 | 회담, 금리, 관세, 규제, 재정정책이 어느 방향으로 전개될지 나눠 본다. |
| 산업 테마 | 구조 성장, 과열, 병목 이동, 2차 수혜 확산 가능성을 나눠 본다. |
| 개별 종목 | 실적 통과, 기대 미달, 밸류에이션 부담, 추세 붕괴 가능성을 나눠 본다. |
| 포트폴리오 | 공격, 유지, 방어, 현금 확보, 다른 축 보강 여부를 나눠 본다. |

## 기본 원칙

### 1. 점 예측 금지

`오른다` 또는 `내린다` 하나로 결론 내리지 않는다. 자산제곱식 사고에서 중요한 것은 결과를 맞히는 것이 아니라, 결과별 대응을 미리 준비하는 것이다.

잘못된 질문:

- `이번 주에 오를까?`
- `이 종목 사도 돼?`
- `이 이벤트는 호재야 악재야?`

올바른 질문:

- `상승 지속, 눌림, 하락 전환 중 어떤 경우의 수가 있는가?`
- `각 경우에 어떤 신호가 나오면 무엇을 할 것인가?`
- `내가 틀렸다는 것을 무엇으로 확인할 것인가?`

### 2. 3개를 기본으로, 복잡하면 5개까지

시나리오는 너무 많아지면 실행할 수 없다. 기본은 3개다.

권장 구조:

| 개수 | 사용 상황 |
|------|-----------|
| 3개 | 일반적인 시장 판단, 종목 판단, 이벤트 대응 |
| 4개 | 핵심 변수 2개를 2x2로 나눌 때 |
| 5개 | 구조 변화, 정책 변수, 지정학 리스크가 동시에 얽힌 복잡한 상황 |

5개를 넘기면 실행 문서가 아니라 연구 노트가 된다. 에이전트 문서에서는 5개 초과 시나리오를 금지한다.

### 3. 시나리오는 경로가 있어야 한다

시나리오는 단순한 방향 예측이 아니다. 반드시 `왜 그렇게 되는지`와 `어떤 순서로 진행되는지`를 포함해야 한다.

필수 구성:

| 항목 | 설명 |
|------|------|
| 출발점 | 현재 시장이 놓인 구조 |
| 핵심 변수 | 방향을 바꿀 수 있는 2~3개 변수 |
| 경로 | 정책, 실적, 수급, 심리, 가격이 연결되는 순서 |
| 결과 | 해당 경로가 현실화될 때 나타날 시장 상태 |
| 대응 | 그때 해야 할 행동 |

### 4. 확률은 감이 아니라 근거의 층으로 배정한다

확률은 확신의 크기가 아니다. 근거가 몇 층으로 쌓였는지를 표시하는 숫자다.

확률 검증 질문:

1. 과거에 비슷한 사례가 있었는가?
2. 정책, 기업, 투자자 등 주요 행위자의 인센티브가 같은 방향인가?
3. 실행에 필요한 돈, 기술, 인력, 제도, 시간이 있는가?
4. 타이밍이 맞는가?
5. 시장이 아직 충분히 반영하지 않았는가?

확률 배정 기준:

| 검증 결과 | 확률 범위 | 해석 |
|-----------|-----------|------|
| 5개 모두 충족 | 50~60% | 주 시나리오 가능 |
| 3~4개 충족 | 30~40% | 보조 시나리오 가능 |
| 2개 이하 충족 | 20% 이하 | 경계 또는 꼬리 시나리오 |

주의:

- 단일 시나리오에 70% 이상을 쉽게 부여하지 않는다.
- 100% 확률은 금지한다.
- 확률 합계는 100%가 되도록 조정한다.
- 확률은 새 정보가 나오면 업데이트한다.

### 5. 트리거는 미리 정한다

시나리오는 트리거가 없으면 해석 놀이가 된다. 트리거는 특정 시나리오가 현실화되고 있음을 보여주는 조기 경보 신호다.

트리거 유형:

| 유형 | 예시 |
|------|------|
| 가격 | 지지선 방어, 상단 돌파, 갭 하락 후 회복, 200일선 회복 |
| 실적 | 가이던스 상향, 수주잔고 증가, 마진 개선, 기대치 미달 |
| 정책 | 금리 방향, 관세 완화, 규제 강화, 재정 지출 |
| 수급 | 기관 매수, ETF 유입, 공매도 감소, 거래량 증가 |
| 심리 | 악재 둔감, 호재 민감, 과열, 패닉 |
| 구조 | 병목 이동, 공급 부족, CAPEX 확대, 리쇼어링 |

### 6. 대응은 시나리오별로 달라야 한다

모든 시나리오의 결론이 `매수`라면 시나리오가 아니다. 시나리오별로 매수, 홀딩, 매도·감축, 현금 유지, 조건부 대기, 헷지가 달라야 한다.

대응 분류:

| 대응 | 의미 |
|------|------|
| 매수 | 조건이 충족되어 실행 문서로 넘길 수 있다. |
| 정찰병 | 아이디어 관찰권을 사는 수준이며, 추가매수 전 사후검증이 필요하다. |
| 보류 | 방향은 맞지만 가격, 이벤트, 실적, 비중 조건이 부족하다. |
| 감축 | 시나리오가 약화되거나 포트폴리오 쏠림이 커졌다. |
| 현금 유지 | 변동성, 이벤트, 가격 위치가 불리하다. |
| 헷지 | 꼬리 리스크가 낮은 확률이지만 충격이 크다. |

시나리오 승격 원칙:

- 시나리오가 `매수`로 분류되려면 트리거, 가격 구간, 무효화 기준이 있어야 한다.
- 트리거가 발생했고 가격 구간이 들어왔으며 포트폴리오 비중이 허용되면 `보류`가 아니라 매수게이트로 보낸다.
- 시나리오가 말한 매수 구간이 실제로 왔는데 계속 대기만 하면 시나리오 운용 실패로 검증한다.

### 7. 무효화 기준은 가격과 가설을 분리한다

시나리오가 틀렸는지 확인하는 기준은 두 가지다.

| 기준 | 의미 |
|------|------|
| 가격 기준 | 차트상 지지선, 추세선, 이동평균, 이전 저점 등이 깨졌는지 확인한다. |
| 가설 기준 | 투자 아이디어의 전제가 무너졌는지 확인한다. |

가격이 버텨도 가설이 깨지면 보류하거나 줄인다. 가설이 살아 있어도 가격이 무너지면 수량과 비중을 통제한다.

### 8. 시나리오는 매주 업데이트한다

시나리오는 고정 문서가 아니다. 새 칼럼, 실적, 이벤트, 가격 반응이 나오면 확률과 트리거를 업데이트해야 한다.

업데이트 원칙:

- 새 정보가 기존 시나리오를 강화하는지 확인한다.
- 새 정보가 기존 시나리오를 약화하는지 확인한다.
- 새로운 시나리오가 필요한지 확인한다.
- 일시 이벤트인지 구조 변화인지 분리한다.
- 실행 결론은 별도 실행 문서로 넘긴다.

## 작성 절차

### 1단계: 질문을 바꾼다

사용자의 질문이 `살까 말까`라면, 먼저 `어떤 시나리오에서 살 수 있는가`로 바꾼다.

변환 예시:

| 원 질문 | 시나리오 질문 |
|---------|---------------|
| 오늘 매수해? | 오늘 매수가 가능한 시나리오는 무엇이고, 어떤 조건이 충족됐는가? |
| 이 종목 괜찮아? | 이 종목이 좋아지는 경로와 깨지는 경로는 무엇인가? |
| 이벤트 끝나면 올라? | 이벤트 결과별로 자금 흐름과 가격 반응은 어떻게 달라지는가? |
| 지금 상승장이야? | 상승 지속, 과열 조정, 추세 붕괴 중 어느 시나리오가 우세한가? |

### 2단계: 시간축을 나눈다

하나의 시나리오를 단기, 중기, 장기로 섞어 쓰면 판단이 흔들린다.

| 시간축 | 질문 |
|--------|------|
| 단기 | 오늘 또는 이번 주에 실행할 수 있는가? |
| 중기 | 1~3개월 안에 비중을 늘릴 구조인가? |
| 장기 | 1년 이상 유지할 구조 변화인가? |

장기 시나리오가 좋아도 단기 매수는 아닐 수 있다. 단기 반등이 가능해도 장기 구조가 약하면 정찰병 이상으로 가지 않는다.

### 3단계: 핵심 변수 2~3개를 고른다

변수가 많으면 결론이 흐려진다. 시장을 실제로 움직일 변수만 남긴다.

대표 변수:

- 금리와 유동성
- 경기와 실적
- 정책과 규제
- 가격 위치와 수급
- 밸류에이션과 기대치
- 포트폴리오 쏠림과 현금 여력

### 4단계: 시나리오를 만든다

기본형:

| 구분 | 설명 |
|------|------|
| 긍정 시나리오 | 구조가 강화되고 가격이 따라오는 경우 |
| 중립 시나리오 | 방향은 맞지만 시간이 필요하거나 가격이 쉬어가는 경우 |
| 부정 시나리오 | 핵심 가설이 약화되거나 가격 구조가 무너지는 경우 |

복합형:

| 변수 A | 변수 B | 시나리오 |
|--------|--------|----------|
| 우호적 | 우호적 | 공격 가능 |
| 우호적 | 비우호적 | 선별 또는 정찰 |
| 비우호적 | 우호적 | 반등 가능하나 확인 필요 |
| 비우호적 | 비우호적 | 방어 또는 보류 |

### 5단계: 확률을 배정한다

각 시나리오마다 근거 검증 질문을 적용한다. 확률은 감정이 아니라 근거의 개수와 강도로 정한다.

출력 규칙:

- 주 시나리오 1개를 정한다.
- 보조 시나리오 1~2개를 정한다.
- 꼬리 리스크를 최소 1개 둔다.
- 각 시나리오의 확률 합계는 100%로 맞춘다.

### 6단계: 트리거와 무효화 기준을 붙인다

각 시나리오에는 반드시 아래 항목이 있어야 한다.

| 항목 | 설명 |
|------|------|
| 강화 트리거 | 이 신호가 나오면 해당 시나리오 확률을 올린다. |
| 약화 트리거 | 이 신호가 나오면 해당 시나리오 확률을 낮춘다. |
| 무효화 기준 | 이 기준이 나오면 해당 시나리오를 폐기한다. |

### 7단계: 대응을 붙인다

시나리오별 대응은 실행 가능한 문장이어야 한다.

나쁜 예:

- `좋게 본다.`
- `관심을 둔다.`
- `기회가 있을 수 있다.`

좋은 예:

- `지지선 방어 후 거래량이 붙으면 정찰병 가능.`
- `실적 전에는 보류하고, 발표 후 가이던스와 가격 반응을 확인한다.`
- `상단 돌파 실패 후 2거래일 이상 밀리면 추가매수 금지.`
- `기존 보유 축과 중복되면 신규 매수보다 비중 조절을 우선한다.`

## 칼럼 업데이트 처리 규칙

총괄 브레인이 새 칼럼 요청으로 분류한 뒤, 새 칼럼은 실무상 먼저 [칼럼 인입/반영/분석](<../01_칼럼인입/자산제곱_칼럼인입_반영_분석.md>) 에이전트를 통과한다. 날짜가 있는 이벤트와 단기 변수는 [이벤트/시나리오 업데이트](<../02_이벤트시나리오/자산제곱_이벤트시나리오.md>)에서 먼저 관리하고, 이 문서는 장기 경로와 확률 구조가 바뀌는 내용만 처리한다.

1. 기존 철학을 바꾸는 내용인지 확인한다.
2. 기존 시나리오의 확률만 바꾸는 내용인지 확인한다.
3. 새로운 핵심 변수를 추가해야 하는지 확인한다.
4. 일시 이벤트인지 구조 변화인지 분리한다.
5. 실행 가능한 내용만 실행 문서로 넘긴다.
6. 긴 근거는 근거 인덱스나 주간기록으로 보낸다.

분류 기준:

| 칼럼 내용 | 처리 |
|-----------|------|
| 장기 구조 변화 | 시나리오의 핵심 변수 또는 장기 경로에 반영 |
| 이벤트 변수 | 02 이벤트/시나리오 업데이트와 이벤트 문서에 등록 |
| 종목 언급 | 종목 후보군으로 분류하고 매수 게이트 전에는 실행 금지 |
| 가격 조건 | 시나리오 트리거로 반영 |
| 원칙 변화 | 철학 또는 판단원칙 문서 업데이트 검토 |

## 오역 방지

| 오역 | 올바른 해석 |
|------|-------------|
| 시나리오 확률이 높다 = 바로 매수 | 확률이 높아도 가격, 비중, 이벤트 조건을 통과해야 한다. |
| 긍정 시나리오가 있다 = 낙관 | 긍정, 중립, 부정, 꼬리 리스크를 동시에 보는 것이다. |
| 부정 시나리오가 있다 = 아무것도 하지 않는다 | 부정 시나리오는 수량 조절, 현금, 헷지, 보류 기준을 제공한다. |
| 트리거가 나왔다 = 무조건 실행 | 트리거 후에도 포트폴리오 쏠림과 매수 게이트를 확인해야 한다. |
| 새 칼럼이 나왔다 = 새 문서 생성 | 기존 시나리오의 변수, 확률, 트리거를 업데이트하는 것이 우선이다. |

## 정합성 검증

| 검증 항목 | 판정 |
|-----------|------|
| 철학 문서와의 정합성 | 적합. 점 예측 금지, 복수 시나리오, 분할 대응을 하위 구조로 번역했다. |
| 시나리오 기반 추론 방법과의 정합성 | 적합. 3~5개 시나리오, 확률, 트리거, 주간 업데이트를 반영했다. |
| 시나리오 분석 방법론과의 정합성 | 적합. 핵심 변수 2~3개, 2x2 사고, 구조 해석을 반영했다. |
| 기술적 분석 프로젝트와의 정합성 | 적합. 가격, 시간, 구조, 유동성을 트리거와 무효화 기준으로 연결했다. |
| 실행 문서와의 분리 | 적합. 이 문서에는 종목 추천, 매수 가격, 이번 주 매매 판단을 넣지 않는다. |

최종 판정:

- 이 문서는 철학 문서의 하위 문서로 적합하다.
- 시나리오를 시장 전망이 아니라 `대응 체계`로 정의했기 때문에 문서 혼선을 줄일 수 있다.
- 다음 문서는 `판단원칙`이 적합하다. 판단원칙은 이 시나리오를 실제 운용 판단으로 번역하는 역할을 맡아야 한다.
