# 자산제곱 판단원칙

> 목적: 자산제곱식 철학과 시나리오를 실제 투자 판단 기준으로 번역한다.
> 작성일: 2026-05-23
> 상위/입력 문서: [자산제곱 총괄 브레인](<../00_총괄브레인/자산제곱_총괄브레인.md>), [자산제곱 칼럼 인입/반영/분석](<../01_칼럼인입/자산제곱_칼럼인입_반영_분석.md>), [자산제곱 이벤트/시나리오 업데이트](<../02_이벤트시나리오/자산제곱_이벤트시나리오.md>), [자산제곱 철학](<../03_철학/자산제곱_철학.md>), [자산제곱 시나리오 사고](<../04_시나리오/자산제곱_시나리오_사고.md>), [자산제곱 종목 아카이브](<../05_종목아카이브/자산제곱_종목아카이브.md>)
> 하위 문서: [자산제곱 매수게이트](<../07_매수게이트/자산제곱_매수게이트.md>)
> 범위: 이 문서는 종목 추천, 오늘 매매, 가격 조건, 보유 수량을 쓰는 문서가 아니다.
> 금지: 특정 종목을 `매수`, `매도`, `몇 주`로 결론 내리지 않는다. 실행 판단은 별도 `매수게이트`와 `이번주_실행판`에서 한다.

## 한 줄 정의

자산제곱식 판단원칙은 `좋은 이야기`를 바로 매수로 번역하지 않고, 구조적 방향, 종속 변수, 기업 품질, 가격 좌표, 포트폴리오 충돌, 무효화 기준을 통과했을 때만 실행 후보로 승격하는 필터다.

출처 구분:

- 구조, 종속 변수, 정보 연결, 가격 좌표, 포트폴리오, 무효화 기준은 자산제곱 프로젝트와 투자 전략 칼럼을 종합한 `칼럼 종합`이다.
- `관찰/후보/조건부 실행 후보/실행 후보` 분류명과 6축 표는 실행을 위해 만든 `에이전트 운용규칙`이다.
- 계좌 번호 배정, 1주 정찰병, 현재 손익 트리거는 `포트폴리오 입력`이다.

## 문서 위치

| 문서 | 역할 |
|------|------|
| 총괄 브레인 | 사용자 요청을 분류하고 판단원칙 호출 여부를 정한다. |
| 칼럼 인입/반영/분석 | 새 칼럼을 읽고 반영 위치를 결정한다. |
| 이벤트/시나리오 업데이트 | 이벤트와 단기 변수의 등록, 추적, 종료를 관리한다. |
| 철학 | 왜 그렇게 사고해야 하는가 |
| 시나리오 | 가능한 미래를 어떻게 나누는가 |
| 종목 아카이브 | 종목의 출처와 언급 근거를 보관한다. |
| 판단원칙 | 무엇을 확인해야 판단이 성립하는가 |
| [매수게이트](<../07_매수게이트/자산제곱_매수게이트.md>) | 오늘 또는 이번 주에 살 수 있는가 |
| 실행판 | 실제로 무엇을 사고, 보류하고, 줄일 것인가 |

판단원칙은 `좋다/나쁘다`를 가르는 문서다. `오늘 산다/안 산다`는 하위 실행 문서에서만 판단한다.

## 기본 원칙

### 0. 목적은 수익이고, 기준은 수익을 지키기 위한 장치다

판단원칙은 매수를 막기 위한 체크리스트가 아니다. 자산제곱 칼럼 원문/종합에서 확인한 원칙에 맞는 수익 기회를 찾고, 틀렸을 때 계좌가 망가지지 않도록 크기를 조절하는 필터다.

판단 순서:

1. 큰 흐름과 자산제곱 시나리오가 살아 있는가?
2. 언급 종목이나 섹터가 구조적 수익 기회와 연결되는가?
3. 지금 참여하지 않으면 상승장 소외가 커지는가?
4. 추격 대량매수가 아니라 정찰병/분할로 들어갈 수 있는가?
5. 틀렸을 때 무효화 기준이 있는가?

위 조건이 대부분 맞으면 `매수 없음`으로 닫지 말고 `조건부 실행 후보`로 올린다. 방어적 판단은 필요하지만, 상승장 참여를 계속 놓치는 것도 판단 오류다.

### 1. 언급과 매수는 다르다

칼럼이나 텔레그램에서 종목이 언급되어도 그것은 판단의 시작점일 뿐이다. 언급 종목은 먼저 아래 단계로 분류한다.

| 등급 | 의미 |
|------|------|
| 관찰 대상 | 구조 설명에 등장했지만 아직 기업 검증이 부족하다. |
| 후보군 | 구조, 섹터, 기업 품질 중 일부가 맞아 추가 분석 가치가 있다. |
| 실행 후보 | 구조, 기업, 가격, 비중, 무효화 기준이 모두 통과했다. |
| 제외 | 테마는 맞아도 가격, 재무, 구조, 포트폴리오 조건이 맞지 않는다. |

### 2. 조건 충족 후보는 매수대기로 승격한다

판단원칙의 목적은 후보를 오래 보관하는 것이 아니라, 자산제곱 칼럼 원문/종합 원칙을 통과한 대상을 실행 경로로 올릴지 결정하는 것이다.

```text
자산제곱 직접/반복 언급
→ 구조·종속 변수·기업 품질·포트폴리오 역할 부합
→ 가격 좌표와 무효화 기준 존재
→ 조건부 실행 후보 또는 실행 후보로 승격
```

승격 기준:

| 상태 | 분류 | 다음 행동 |
|------|------|-----------|
| 구조만 좋고 기업/가격 검증 부족 | 관찰 | 종목_종합정리 |
| 6축 중 일부 통과, 실행 조건 부족 | 후보 | 보강 후 재판정 |
| 6축 대부분 통과, 가격/이벤트/비중 트리거만 남음 | 조건부 실행 후보 | 매수게이트 |
| 6축 통과, 수량과 무효화만 정하면 됨 | 실행 후보 | 매수게이트 또는 실행판 |

`좋은 후보`를 계속 `후보군`으로만 남기는 것은 판단이 아니다. 가격, 이벤트, 비중, 무효화 조건이 충족되면 매수게이트로 올려야 한다.

### 3. 장기 방향과 오늘 매수는 분리한다

장기 테마가 강해도 오늘 매수할 수 없는 경우가 많다. 반대로 단기 눌림이 와도 장기 구조가 약하면 정찰병 이상으로 가면 안 된다.

판단 순서:

1. 장기 구조가 살아 있는가?
2. 중기 흐름이 악화 중인가, 회복 중인가?
3. 단기 가격 위치가 매수 가능한가?
4. 지금 사면 포트폴리오 위험이 커지는가 줄어드는가?

### 4. 계좌 역할을 먼저 정한다

자산제곱 계좌 분리 기준에서는 같은 종목도 `장기 계좌`와 `스윙 계좌`에서 판단이 달라질 수 있다. 따라서 판단원칙은 종목을 좋다/나쁘다로만 분류하지 않고, 어느 계좌에 들어갈 판단인지 먼저 태그한다.

| 계좌 역할 | 핵심 질문 | 판단 기준 |
|-----------|-----------|-----------|
| 장기/코어 | 기업 성장 스토리와 장기 구조가 살아 있는가? | 최소 1년 이상 보유할 각오, 단기 변동보다 시나리오 유효성 우선 |
| 스윙/전술 | 시나리오, 가격, 이벤트, 무효화가 맞는가? | 20~80% 비중 조절 범위 안에서 저점/과매도, 고점/과매수 대응 |
| 현금/방어 | 기회가 왔을 때 쓸 선택권이 충분한가? | 예수금과 단기채는 대기 자산, 장기채는 방어 자산으로 분리 |

`20~80%`는 자산제곱의 비중 조절 칼럼에 직접 근거가 있는 수치다. 다만 실제 계좌번호를 장기/코어와 스윙/전술로 배정하는 것은 현재 사용자 계좌에 맞춘 `포트폴리오 입력`이다.

계좌 역할 태그가 없으면 `실행 후보`로 올리지 않는다. 계좌 역할과 실제 보유 여부는 [포트폴리오 실제 보유현황](<../../../1_브리핑/포트폴리오_브리핑/00_현황/실제_보유현황.md>)을 따른다.

금지:

- 스윙 손실 종목을 근거 없이 장기 보유로 전환하지 않는다.
- 장기 계좌의 수익 종목을 스윙 손실 보전용으로 팔지 않는다.
- 스윙 계좌의 20~80% 비중 규칙을 전체 계좌에 섞어 적용하지 않는다.

### 5. 판단은 최소 6축을 통과해야 한다

하나의 축만 강하면 후보군에 그친다. 실행 후보가 되려면 최소 6축이 모두 설명되어야 한다.

| 축 | 핵심 질문 |
|----|-----------|
| 구조적 방향 | 이 기업이 속한 흐름은 일시 테마인가, 구조 변화인가? |
| 종속 변수 | 이 기업의 실적과 주가를 움직이는 선행 변수는 무엇인가? |
| 기업 품질 | 실제로 돈을 벌고 버틸 수 있는 기업인가? |
| 가격 좌표 | 지금 가격은 기대 대비 유리한 위치인가? |
| 포트폴리오 충돌 | 이미 가진 축과 중복되거나 쏠림을 키우지 않는가? |
| 무효화 기준 | 무엇이 나오면 판단이 틀렸다고 인정할 것인가? |

## 판단 6축

### 1축. 구조적 방향

먼저 종목이 아니라 구조를 본다. 자산제곱식 판단에서 기업은 구조 변화의 표현이다.

확인 질문:

- 이 흐름은 정책, 자본지출, 공급 부족, 기술 전환, 지정학, 인구 구조 중 어디에서 출발하는가?
- 최종 수혜주인가, 병목 해결사인가?
- 단일 내러티브인가, 여러 독립 수요가 겹친 복합 내러티브인가?
- AI, 리쇼어링, 에너지, 방산, 원자재, 바이오, 우주, 주권 AI 중 어디에 연결되는가?
- 구조가 약해져도 다른 수요 축이 받쳐줄 수 있는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 구조 변화가 명확하고, 병목 또는 핵심 연결고리에 위치한다. |
| 조건부 | 구조는 맞지만 아직 실행 데이터나 가격 확인이 부족하다. |
| 탈락 | 뉴스성 테마이거나 최종재 기대만 있고 병목/현금흐름이 약하다. |

### 2축. 종속 변수

좋은 기업도 무엇에 종속되는지 모르면 판단할 수 없다. 종속 변수는 실적 발표 전에 기업 방향을 미리 읽는 선행 지표다.

확인 질문:

- 매출은 어떤 가격, 주문, 수주잔고, 정책, 금리, 원자재, 고객 CAPEX에 종속되는가?
- 선행 지표가 좋아지고 있는가, 악화되고 있는가?
- 고객사가 투자를 줄이면 이 기업도 같이 흔들리는가?
- 원가 상승을 가격에 전가할 힘이 있는가?
- 실적 전후에 확인해야 할 변수는 무엇인가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 핵심 변수 2~3개가 명확하고 최근 방향이 우호적이다. |
| 조건부 | 변수는 알지만 방향이 아직 불확실하다. |
| 탈락 | 무엇에 의해 좋아지고 나빠지는지 설명할 수 없다. |

### 3축. 기업 품질

테마가 맞아도 기업이 약하면 오래 버틸 수 없다. 판단원칙에서 기업 품질은 `성장 스토리`보다 `현금흐름과 생존력`을 먼저 본다.

확인 질문:

- 순이익보다 영업현금흐름과 FCF가 실제로 따라오는가?
- FCF 마진, 안정성, 순이익 대비 현금흐름 전환율이 양호한가?
- 매출 성장과 마진 개선이 같이 일어나는가?
- 부채 만기, 이자 부담, 순부채/EBITDA가 감당 가능한가?
- ROE, ROIC가 높은 이유가 경쟁력 때문인가, 과도한 레버리지 때문인가?
- 고객 집중도, 공급망 리스크, 규제 리스크는 감당 가능한가?
- 경쟁우위가 숫자와 사업 구조로 설명되는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 구조 성장, FCF, 재무 안정성, 경쟁우위가 함께 확인된다. |
| 조건부 | 성장성은 있으나 현금흐름, 밸류에이션, 부채 중 하나가 약하다. |
| 탈락 | 테마만 있고 현금흐름이 없거나 부채/희석/고객 집중 위험이 크다. |

### 4축. 가격 좌표와 기대치

좋은 기업도 비싸게 사면 시간이 필요하다. 가격은 시장 기대와 포지셔닝이 압축된 정보다.

확인 질문:

- 좋은 뉴스가 이미 가격에 반영되었는가?
- 나쁜 뉴스에도 주가가 버티는가, 좋은 뉴스에도 빠지는가?
- 실적은 절대적으로 좋았는가, 기대치 대비 좋았는가?
- 차트상 지지, 저항, 매물대, 추세, 유동성 위치가 설명되는가?
- 지금 진입하면 손절 또는 재검토 좌표를 둘 수 있는가?
- 신고가 추격인지, 눌림 확인인지, 추세 전환인지 구분되는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 기대치 대비 여지가 있고, 차트 좌표와 무효화 위치가 있다. |
| 조건부 | 기업은 좋지만 가격이 앞서 있거나 이벤트 직전이다. |
| 탈락 | 호재가 과도하게 반영되었고 손절/재검토 좌표가 없다. |

### 5축. 포트폴리오 충돌

판단은 개별 종목 안에서 끝나지 않는다. 좋은 종목도 이미 가진 축과 겹치면 위험을 키울 수 있다.

확인 질문:

- 현재 포트폴리오에서 같은 방향으로 움직이는 자산이 이미 많은가?
- 이 종목은 빈칸을 채우는가, 기존 쏠림을 더 키우는가?
- 전력/AI/반도체/방산/에너지/금/코인 등 어느 바구니에 들어가는가?
- 한 종목 또는 한 테마의 하락이 전체 계좌를 흔들 정도인가?
- 현금 또는 단기채를 줄여서 살 만큼 우선순위가 높은가?
- ETF로 대체하는 편이 더 효율적인가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 포트폴리오의 빈칸을 채우거나 위험 분산에 기여한다. |
| 조건부 | 좋은 후보지만 기존 쏠림과 일부 중복된다. |
| 탈락 | 이미 과도하게 가진 축을 더 키운다. |

### 6축. 무효화 기준과 사후검증

매수 전에는 반드시 틀렸을 때의 기준이 있어야 한다. 손절은 단순 가격 손실이 아니라 논리 훼손 여부와 함께 판단한다.

확인 질문:

- 어떤 가격 구조가 깨지면 판단을 재검토할 것인가?
- 어떤 실적, 수주, 마진, 정책, 수급 변화가 나오면 가설이 무너지는가?
- 1차 매수 후 추가매수 전에 무엇을 확인할 것인가?
- 수익이 났을 때 분할 익절 또는 비중 조절 기준은 있는가?
- 보유 이유가 논리인가, 감정인가?
- 시나리오가 살아 있는데 가격만 흔들리는 것인가, 시나리오 자체가 훼손된 것인가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 가격 기준과 가설 기준이 모두 정의되어 있다. |
| 조건부 | 방향은 맞지만 무효화 기준이 불명확하다. |
| 탈락 | 틀렸을 때 무엇을 할지 정해져 있지 않다. |

## 최종 분류

6축을 점검한 뒤 종목이나 자산은 아래 중 하나로만 분류한다.

| 분류 | 조건 | 다음 문서 |
|------|------|-----------|
| 제외 | 구조, 기업, 가격, 포트폴리오 중 치명 결함이 있다. | 기록만 남긴다. |
| 관찰 | 구조는 있으나 기업/가격 검증이 부족하다. | 종목_종합정리 |
| 후보 | 6축 중 4개 이상 통과했지만 실행 조건이 부족하다. | 종목_종합정리, 근거 인덱스 |
| 조건부 실행 후보 | 6축 대부분 통과, 단 가격/이벤트/비중 확인이 남았다. | [매수게이트](<../07_매수게이트/자산제곱_매수게이트.md>) |
| 실행 후보 | 6축 통과, 수량과 조건만 정하면 된다. | 이번주_실행판 |
| 보유 재검토 | 기존 보유 종목의 가설 또는 비중이 흔들린다. | 실제_보유현황, 이번주_실행판 |

## 판단 절차

### 1단계. 칼럼 입력 분류

새 칼럼이 들어오면 바로 종목을 뽑지 않는다. 총괄 브레인이 새 칼럼 요청으로 분류한 뒤, 먼저 [칼럼 인입/반영/분석](<../01_칼럼인입/자산제곱_칼럼인입_반영_분석.md>) 에이전트에서 칼럼의 성격을 분류한다. 판단원칙은 그 결과 판단 기준 변화가 있다고 분류된 내용만 처리한다.

| 칼럼 성격 | 반영 위치 |
|-----------|-----------|
| 철학 변화 | 철학 문서 |
| 시나리오 변화 | 시나리오 문서 |
| 판단 기준 변화 | 판단원칙 문서 |
| 새로운 종목 후보군 | 종목_종합정리 |
| 이번 주 이벤트 | 이벤트 문서 또는 실행판 |
| 오늘 매매 조건 | 매수게이트와 실행판 |

### 2단계. 종목 언급을 구조로 번역

종목명을 그대로 매수 후보로 옮기지 않는다.

번역 예시:

| 언급 | 먼저 번역할 구조 |
|------|----------------|
| 전력기기 | 데이터센터 전력 병목, 노후 송전망, 변압기 부족 |
| 방산 | 지정학 장기화, 국가안보 예산, 공급망 재편 |
| 한국 반도체 | HBM, 메모리 가격, 빅테크 CAPEX, 글로벌 AI 인프라 |
| 에너지 | 유가, 천연가스, 전력 수요, 지정학 헷지 |
| 피지컬 AI | 로봇, 센서, 액추에이터, 자동화, 노동 병목 |
| 리쇼어링 | 전력, 자동화, 방산, 패키징, 소재, 인프라 병목 |

### 3단계. 6축 점검표 작성

모든 후보는 아래 형식을 따른다.

| 축 | 판정 | 근거 | 보류/탈락 이유 |
|----|------|------|----------------|
| 구조적 방향 |  |  |  |
| 종속 변수 |  |  |  |
| 기업 품질 |  |  |  |
| 가격 좌표 |  |  |  |
| 포트폴리오 충돌 |  |  |  |
| 무효화 기준 |  |  |  |

### 4단계. 시간축 태그

판단에는 반드시 시간축을 붙인다.

| 태그 | 의미 |
|------|------|
| 단기 | 오늘 또는 이번 주 가격/이벤트 대응 |
| 중기 | 1~3개월 실적, 수급, 정책 확인 |
| 장기 | 1년 이상 구조 변화 |
| 혼합 | 장기 구조는 좋지만 단기 가격 조건이 필요함 |

### 5단계. 실행 문서로 넘길지 결정

아래 조건을 모두 만족할 때만 매수게이트로 넘긴다.

- 구조적 방향이 통과 또는 강한 조건부다.
- 종속 변수 2~3개가 확인되어 있다.
- 기업 품질에 치명 결함이 없다.
- 가격 좌표와 무효화 기준이 있다.
- 포트폴리오 쏠림을 악화시키지 않는다.
- 단기/중기/장기 중 어떤 시간축 판단인지 명확하다.

## 오판 방지 규칙

| 오판 | 방지 규칙 |
|------|-----------|
| 언급 종목을 모두 사려 한다 | 언급은 관찰 대상이며 6축 통과 전 매수 후보가 아니다. |
| 좋은 실적만 보고 산다 | 기대치 대비 결과와 발표 후 가격 반응을 함께 본다. |
| 조정이라는 이유만으로 산다 | 구조 유지, 지지 확인, 무효화 위치가 있어야 한다. |
| 장기 테마를 오늘 매수로 오역한다 | 장기 구조와 단기 가격 좌표를 분리한다. |
| 포트폴리오 쏠림을 무시한다 | 같은 방향 자산이면 좋은 종목도 보류할 수 있다. |
| 가격만 보고 손절한다 | 가격 기준과 가설 기준을 분리해 판단한다. |
| 수익이 났다고 무조건 판다 | 비중, 시나리오 생존 여부, 과열 신호를 함께 본다. |
| 현금이 아까워서 산다 | 현금은 선택권이며, 조건 미충족 매수는 금지한다. |

## 정합성 검증 기록

### 1차 검증: 철학/시나리오와의 경계

| 항목 | 판정 | 내용 |
|------|------|------|
| 철학 반복 여부 | 통과 | 철학의 상위 문장을 반복하지 않고, 판단 기준으로 번역했다. |
| 시나리오 침범 여부 | 통과 | 확률 배정, 트리거 설계 자체는 시나리오 문서에 두고, 이 문서는 판단 통과 기준에 집중했다. |
| 시간축 분리 | 통과 | 단기, 중기, 장기, 혼합 태그를 별도 항목으로 둔다. |
| 무효화 기준 | 통과 | 시나리오의 무효화 개념을 가격 기준과 가설 기준으로 나누어 판단원칙에 반영했다. |

### 2차 검증: 실행 문서 침범 여부

| 항목 | 판정 | 내용 |
|------|------|------|
| 매수 추천 여부 | 통과 | 특정 종목의 매수/매도/수량을 적지 않았다. |
| 매수게이트 침범 여부 | 통과 | 오늘 매수 가능 여부가 아니라 매수게이트로 넘길 조건만 정의했다. |
| 실행판 침범 여부 | 통과 | 실제 주문, 보유 수량, 이번 주 행동은 포함하지 않았다. |
| 포트폴리오 문서 침범 여부 | 통과 | 실제 보유 비중 조정은 다루지 않고, 충돌 여부를 판단하는 원칙만 둔다. |

### 3차 검증: 근거 칼럼 누락 여부

| 축 | 판정 | 근거 |
