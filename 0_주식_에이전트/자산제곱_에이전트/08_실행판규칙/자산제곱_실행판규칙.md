## 한 줄 정의

실행판규칙 에이전트는 매수게이트의 `G0~G5`, `R1` 판정을 실행판의 `매수`, `홀딩`, `매도·감축`, `조건부 대기`로 압축하는 문장 변환 규칙이다.

출처 구분:

- 실행판의 네 블록, G코드 변환표, 조건부 대기 후보 수 제한은 `에이전트 운용규칙`이다.
- `1주`, `전체 자산 1~2%`, 계좌별 수량은 `포트폴리오 입력`이다.
- 실행판 문장은 칼럼 원문을 그대로 옮기는 곳이 아니라, 칼럼 원칙과 매수게이트 판정을 행동 문장으로 번역하는 곳이다.

## 핵심 원칙

### 1. 실행판은 한 화면 판단 문서다

실행판은 분석 문서가 아니다. 사용자가 1~2분 안에 아래 질문에 답할 수 있어야 한다.

1. 오늘 살 것이 있는가?
2. 오늘 팔거나 줄일 것이 있는가?
3. 사면 조건과 수량은 무엇인가?
4. 홀딩이라면 유지 조건과 금지 조건은 무엇인가?
5. 조건부 대기라면 승격 트리거는 무엇인가?

최종 표시는 항상 아래 네 칸으로 끝낸다.

```text
매수:
홀딩:
매도·감축:
조건부 대기:
```

에이전트 내부 기준은 정교해도 되지만, 사용자 화면은 매수/홀딩/매도 판단을 흐리면 안 된다.

### 2. 매수 없음은 반드시 분류한다

`매수 없음`만 쓰면 판단이 흐려진다. 반드시 아래 중 하나로 쓴다.

| 표현 | 의미 |
|------|------|
| `후보 없음` | 게이트에 올릴 후보 자체가 없다. |
| `조건 미충족` | 후보는 있으나 가격, 이벤트, 무효화 중 하나가 부족하다. |
| `추격 금지` | 구조는 맞지만 가격 위치가 나쁘다. |
| `검증 우선` | 이미 산 정찰병 또는 보유 종목의 생존 확인이 먼저다. |
| `정리 우선` | 신규 매수보다 위험 축소, 손절, 정리 검토가 먼저다. |
| `리스크 필터 우선` | 시장 변수 확인 전 추가매수 금지다. |

### 3. 조건부 대기는 조건이 없으면 실행판에 올리지 않는다

조건부 대기 후보에는 반드시 조건, 수량, 무효화가 있어야 한다.

| 필수 항목 | 예시 |
|-----------|------|
| 조건 | 실적 후 눌림 방어, 이벤트 종료, 돌파선 재지지 |
| 수량 | 1주, 소액 정찰병, 전체 자산 1~2% |
| 무효화 | 저점 재이탈, 실적 가설 훼손, 유가 급락 지속 |
| 다음 확인 | 장 마감 후, 실적 발표 후, 회담 종료 후 |

조건이 없으면 실행판에 올리지 않고 근거 인덱스나 종목 정리에만 남긴다.

수량 예시는 자산제곱 원문 공식이 아니라 현재 계좌 규모에 맞춘 `포트폴리오 입력`이다.

### 4. 조건이 충족된 후보는 실행 문장으로 쓴다

실행판규칙은 `보류` 문장을 만드는 문서가 아니라, 매수게이트 판정을 실제 행동 문장으로 바꾸는 문서다. `G3/G4/G5` 판정이 내려진 대상은 근거 기록에 숨기지 않고 실행판 네 블록에 올린다.

| 게이트 | 실행판 처리 |
|--------|-------------|
| `G3` | `조건부 대기`에 조건, 수량, 무효화를 적는다. 조건이 이미 충족됐으면 `매수`로 올린다. |
| `G4` | `매수`에 `1차 매수 가능`으로 쓴다. |
| `G5` | `매수`에 `추가매수 가능`으로 쓴다. |

금지:

- `G3/G4/G5` 판정을 받은 대상을 단순 `관찰`로 낮추지 않는다.
- 조건이 충족됐는데 `좀 더 보자`, `대기`, `추격 금지`만 쓰지 않는다.
- `후보군`이라는 단어로 실행 판단을 흐리지 않는다. 실행 가능하면 `조건부 대기`, `1차 매수`, `추가매수` 중 하나로 쓴다.

## 입력

실행판규칙은 아래 입력만 받는다.

| 입력 | 출처 |
|------|------|
| 매수게이트 판정 코드 | `G0`, `G1`, `G2`, `G3`, `G4`, `G5`, `R1` |
| 보유 현황 변화 | 실제 보유현황 |
| 포폴 주간 스냅샷 | 실제 보유현황, 주간기록 |
| 이벤트 결과 또는 예정 | 다가올 이벤트 |
| 운용 원칙 변화 | 현재 운용 판단 |
| 긴 근거 요약 | 근거 인덱스 |

입력 없이 실행판에 새 종목을 직접 추가하지 않는다.

## 승격 반영 시점

실행판규칙은 매수게이트가 승격한 후보를 다음 시점에 실행판으로 옮긴다.

| 시점 | 실행판 처리 |
|------|-------------|
| 신규 칼럼 반영 직후 | 새 후보를 `조건부 대기` 또는 근거 기록으로 배치 |
| 주간 보유현황 갱신 직후 | 보유 종목을 `G5`, `R1`, `수익 보호`, `추가매수 금지`로 재배치 |
| 이벤트 종료 후 가격 반응 확인 직후 | 조건 충족 후보를 `G3` 또는 `G4`로 승격해 `조건부 대기` 또는 `매수`에 반영 |
| 가격 트리거 도달 확인 직후 | 조건부 후보를 실행 가능한 `G3` 문장으로 변환 |
| 사용자 요청으로 특정 후보를 재심사한 직후 | 결과를 `매수`, `홀딩`, `매도·감축`, `조건부 대기` 중 하나로 정리 |

실행판은 후보를 상시 감시하지 않는다. 사용자가 최신 가격, 이벤트 결과, 보유 스냅샷을 전달하거나 에이전트 실행을 요청하면 그 시점의 입력으로 승격 여부를 판단한다.

## 게이트 코드 변환표

| 매수게이트 코드 | 실행판 위치 | 버튼 문장 |
|-----------------|-------------|-----------|
| `G0 제외` | 실행판에 올리지 않음 | 필요 시 근거 인덱스만 남김 |
| `G1 관찰` | 원칙적으로 실행판 제외 | 필요 시 `조건부 대기`의 낮은 우선순위로만 남김 |
| `G2 보류` | `조건부 대기` 또는 `홀딩`의 금지 조건 | `조건부 대기. 사유: ... 승격 트리거: ...` |
| `G3 조건부 정찰병` | `조건부 대기` | `조건 충족 시 1주/소액 정찰병. 무효화: ...` |
| `G4 1차 매수 가능` | `매수` | `1차 매수 가능. 수량: ... 무효화: ...` |
| `G5 추가매수 가능` | `매수` | `추가매수 가능. 기존 가설 유지, 재지지 확인.` |
| `R1 정리/감축 검토` | `매도·감축` | `정리/감축 검토. 조건: ...` |

## 포폴 주간 현황 문장화 규칙

사용자가 포폴 주간 현황을 전달한 경우 실행판규칙은 계좌 전체를 장문으로 다시 쓰지 않고, 행동이 필요한 변화만 실행판에 남긴다.

반영 순서:

1. 포트폴리오 에이전트의 `실제_보유현황.md` 스냅샷 기준일이 먼저 최신화됐는지 확인한다.
2. 대상 계좌를 먼저 붙인다. `6212-7075`는 장기/코어, `6555-6037`은 스윙/전술이다.
3. 현금성, 위험자산, 1~6 역할별 비중 변화는 한 줄 요약으로만 쓴다.
4. 실행판에는 `G3`, `G4`, `G5`, `R1`, `추가매수 금지`, `수익 보호`, `리스크 체크`를 올린다.
5. 변화 없는 보유 종목은 실행판에 반복하지 않는다.
6. 긴 이유는 주간기록이나 근거 인덱스로 보내고 실행판에는 버튼 문장만 남긴다.

표준 문장:

| 상황 | 실행판 문장 |
|------|-------------|
| 계좌 방어력 충분 | `[계좌] 현금성 충분. 신규 대량 매수보다 조건부 진입만 허용.` |
| 위험자산 과소 | `[계좌] 위험자산 부족. 단, G3/G4/G5 조건 충족 대상만 편입.` |
| 특정 축 포화 | `[계좌] 포트폴리오 포화. 같은 축 추가매수 금지.` |
| 신규 조건부 진입 | `[계좌] G3 조건부 정찰병. 조건: ... 수량: ... 무효화: ...` |
| 신규 1차 진입 | `[계좌] G4 1차 매수 가능. 조건: ... 수량: ... 무효화: ...` |
| 기존 보유 추가 가능 | `[계좌] G5 추가매수 가능. 조건: ... 수량: ... 무효화: ...` |
| 기존 보유 정리 검토 | `[계좌] R1 정리/감축 검토. 사유: ... 확인 조건: ...` |
| 손실 보유 물타기 위험 | `[계좌] 추가매수 금지. 가설 보강 전 물타기 금지.` |
| 큰 수익 보유 | `[계좌] 수익 보호. 추가매수보다 방어선 확인.` |

금지 문장:

- `좋아 보이니 더 사도 됨`
- `많이 빠졌으니 물타기 가능`
- `수익 중이니 그냥 둔다`
- `전체적으로 괜찮음`

위 문장은 행동, 조건, 무효화가 없으므로 실행판 문장으로 쓰지 않는다.

## 실행판 표준 구조

`이번주_실행판.md`는 아래 순서를 유지한다.

### 1. 메타데이터

필수 항목:

- 기준일
- 보유 스냅샷 기준일
- 기준 문서
- 반영 칼럼/이벤트
- 매수게이트 적용 여부

### 2. 오늘 결론

가장 위에 둔다. 장문 금지.

필수 문장:

```text
매수:
홀딩:
매도·감축:
조건부 대기:
```

규칙:

- `없음`이면 왜 없는지 한 줄로 붙인다.
- `조건부`면 조건을 바로 붙인다.
- `금지`면 금지 사유를 바로 붙인다.
- 오늘 결론은 최대 4개 블록만 둔다.

### 3. 세부 판단은 네 블록 아래에만 붙인다

실행판의 상단 구조는 네 블록으로 고정한다. 표가 필요하면 해당 블록 아래 보조표로만 붙이고, 별도 상위 섹션으로 `이번 주 액션`, `보류/관찰`, `매수 금지`, `정리 후보`를 만들지 않는다.

#### 매수 보조표

| 대상 | 게이트 | 수량 | 조건 | 무효화 |
|------|--------|------|------|--------|

#### 홀딩 보조표

| 대상 | 유지 이유 | 추가매수 금지/확인 조건 |
|------|-----------|-------------------------|

#### 매도·감축 보조표

| 대상 | 조건 | 행동 | 재검토 기준 |
|------|------|------|-------------|

#### 조건부 대기 보조표

| 대상 | 게이트 | 승격 조건 | 수량 | 무효화 | 다음 확인 |
|------|--------|-----------|------|--------|-----------|

규칙:

- `조건부 대기`는 최대 3개만 둔다.
- 수량과 무효화가 없으면 `조건부 대기`에 넣지 않는다.
- 급등 추격, 실적 직전 대형 진입, 무효화 기준 없음, 포트폴리오 포화, 손실 물타기, 구조 훼손, 리스크 이벤트 미확인은 해당 블록의 금지 조건으로 적는다.
- 리스크 체크는 독립 후보가 아니라 `홀딩`, `매도·감축`, `조건부 대기`의 확인 조건으로 번역한다.
- 마지막 문장은 네 블록의 결론을 반복하지 않는다. 필요한 경우 다음 확인일만 남긴다.

## 금지 문장

실행판에서는 아래 표현을 쓰지 않는다.

| 금지 표현 | 이유 | 대체 표현 |
|-----------|------|-----------|
| `괜찮아 보임` | 버튼이 없다. | `보유 / 보류 / 조건부 / 금지` |
| `좋아 보임` | 조건이 없다. | `조건 충족 시 G3` |
| `굳` | 판단 근거가 없다. | `통과 / 보류 / 금지` |
| `매수 없음` | 이유가 없다. | `조건 미충족으로 매수 없음` |
| `좀 더 보자` | 트리거가 없다. | `다음 트리거: ...` |
| `분위기 좋음` | 자산제곱식 근거가 아니다. | `정합성/가격/이벤트 중 무엇이 통과했는지 명시` |

## 오판 방지

| 오판 | 방지 규칙 |
|------|-----------|
| 실행판이 분석문서가 된다 | 긴 근거는 근거 인덱스로 보낸다. |
| 후보가 너무 많아진다 | 조건부 대기 후보는 최대 3개다. |
| 조건부 대기가 방치된다 | 조건부 대기에는 다음 트리거를 붙인다. |
| 매수 없음이 반복된다 | 매수 없음의 유형을 분류한다. |
| FOMO 때문에 조건을 낮춘다 | 게이트 코드와 무효화 기준 없이는 실행판에 올리지 않는다. |
| 손실 물타기가 실행된다 | 가설 강화 없으면 `홀딩`의 추가매수 금지 조건으로 보낸다. |

## 정합성 검증

### 1차 검증: 매수게이트와의 경계

| 항목 | 판정 | 이유 |
|------|------|------|
| 게이트 재판정 여부 | 통과 | 게이트 판정을 새로 만들지 않고 실행판 배치로만 변환한다. |
| 실행판 이관 기준 | 통과 | G3/G4/G5 또는 R1만 네 블록에 올린다. |
| 보류 후보 처리 | 통과 | G1/G2는 트리거와 무효화가 있을 때만 조건부 대기로 남긴다. |

### 2차 검증: 실행판과의 경계

| 항목 | 판정 | 이유 |
|------|------|------|
| 실제 체결 기록 침범 여부 | 통과 | 체결과 계좌 반영은 실제 보유현황 또는 주간기록에서 한다. |
| 장문 근거 방지 | 통과 | 긴 근거는 근거 인덱스로 보내고 실행판에는 버튼 문장만 남긴다. |
| 한 화면 판단 | 통과 | 오늘 결론을 최상단에 두고 후보 수를 제한한다. |

### 3차 검증: 자산제곱 정합성

| 항목 | 판정 | 이유 |
|------|------|------|
| 좋은 종목과 오늘 매수 분리 | 통과 | G1/G2 후보를 실행 버튼으로 올리지 않는다. |
| 과보수 방지 | 통과 | 매수 없음도 유형과 다음 트리거를 남긴다. |
| FOMO 방지 | 통과 | 무효화, 수량, 조건이 없으면 실행판에 올리지 않는다. |
| 사후검증 가능성 | 통과 | 조건/무효화/다음 확인을 필수 항목으로 둔다. |
