# 자산제곱 검증규칙

> 목적: 실행판의 매수/홀딩/매도·감축/조건부 대기 판단이 자산제곱식 절차를 따랐는지 사후검증하고, 누락·과보수·추격·포화 오류를 다음 판단에 반영한다.

## 한 줄 정의

검증규칙 에이전트는 `무엇을 샀는가`보다 `왜 그렇게 판단했는가`, `무엇을 놓쳤는가`, `다음에는 어떤 규칙을 고칠 것인가`를 기록하는 사후검증 문서다.

출처 구분:

- 결과보다 과정, 놓친 상승 검증, 손실만으로 판단하지 않는 원칙은 `칼럼 종합`이다.
- `누락/과보수/승격 실패/추격/포화/물타기` 오류명과 보정 경로는 `에이전트 운용규칙`이다.
- 검증은 칼럼 원문을 새로 만드는 절차가 아니라, 에이전트 해석이 칼럼 원칙을 벗어났는지 확인하는 절차다.

## 핵심 원칙

### 1. 결과 검증과 과정 검증을 분리한다

좋은 결과가 좋은 판단을 뜻하지 않고, 나쁜 결과가 나쁜 판단을 뜻하지 않는다. 검증은 결과와 과정을 분리한다.

| 구분 | 질문 |
|------|------|
| 과정 검증 | 당시 문서 체계와 규칙을 제대로 통과했는가? |
| 결과 검증 | 이후 가격, 실적, 이벤트 반응이 판단과 맞았는가? |
| 보정 검증 | 틀렸다면 어느 에이전트 규칙을 고쳐야 하는가? |

### 2. 놓친 상승도 검증 대상이다

매수하지 않은 결정도 판단이다. 특히 사용자가 불안해했던 `매수 없음 반복`, `바닥권 정찰병 누락`, `개별주 기회 누락`은 반드시 검증한다.

검증 질문:

- 후보는 문서에 있었는가?
- 있었는데 실행판으로 못 올라왔는가?
- 매수게이트가 너무 보수적이었는가?
- 가격 트리거가 과도하게 늦었는가?
- 실행판 문장이 모호해서 행동으로 이어지지 않았는가?

### 3. 승격 실패를 별도 오류로 본다

자산제곱이 직접 또는 반복 언급한 종목이 원칙과 현재 시나리오에 맞고, 가격·이벤트·비중·무효화 조건까지 충족했는데도 실행판으로 올라가지 않았다면 단순 보류가 아니라 `승격 실패`다.

승격 실패 검증 질문:

- 자산제곱 직접/반복 언급이 있었는가?
- 판단원칙에서 `조건부 실행 후보` 또는 `실행 후보`가 될 근거가 있었는가?
- 매수게이트에서 `G3/G4/G5`가 가능했는데 `G1/G2`로 낮춘 것은 아닌가?
- 실행판에서 `조건부 대기`, `1차 매수`, `추가매수` 대신 모호한 관찰 문장으로 처리했는가?
- 이후 가격이 움직였다면 누락 원인은 가격 예측 실패인가, 프로세스 승격 실패인가?

### 4. 보정은 문서 위치까지 지정한다

검증 결과는 감상으로 끝내지 않는다. 보정 위치를 지정한다.

| 오류 위치 | 보정 문서 |
|-----------|-----------|
| 칼럼 하단/뷰포인트 누락 | 칼럼 인입 |
| 종목 출처 누락 | 종목 아카이브 |
| 좋은 후보 판단 누락 | 판단원칙 |
| 실행 후보 승격 실패 | 매수게이트 |
| 실행 문장 모호 | 실행판규칙 |
| 실제 보유 반영 누락 | 실제 보유현황 |
| 긴 근거 누락 | 근거 인덱스 |

## 검증 시점

| 시점 | 해야 할 검증 |
|------|--------------|
| 주간기록 작성 시 | 실행판 판단이 실제 계좌와 시장 반응에 도움이 됐는지 확인 |
| 실적 발표 후 | 실적 전/후 판단과 가격 반응이 맞았는지 확인 |
| 이벤트 종료 후 | 이벤트 전 시나리오와 실제 결과/가격 반응 비교 |
| 큰 급등 후 | 놓친 종목이 종목_종합정리/후보 판정/게이트/실행판 어디서 막혔는지 확인 |
| 큰 손실 후 | 가설 훼손인지 정상 변동성인지 확인 |
| 새 칼럼 반영 후 | 기존 원칙을 고쳐야 하는지, 단기 실행만 바꿀지 확인 |
| 포폴 주간 현황 반영 후 | 실제 보유현황, 실행판, 주간기록이 같은 스냅샷 기준으로 맞는지 확인 |

## 포폴 주간 스냅샷 검증

포폴 주간 현황은 실행판의 입력 데이터다. 검증규칙은 판단이 맞았는지만 보지 않고, 계좌 스냅샷이 문서 체계에 제대로 반영됐는지도 확인한다.

검증 질문:

| 질문 | 통과 기준 |
|------|-----------|
| 스냅샷 기준일이 있는가? | 포트폴리오 `실제_보유현황.md`, 실행판, 주간기록의 기준일이 일치한다. |
| 총자산, 예수금, 현금성, 위험자산이 반영됐는가? | 예수금과 `SHY` 등 현금성 자산이 구분된다. |
| 계좌 역할이 반영됐는가? | `6212-7075` 장기/코어, `6555-6037` 스윙/전술, 현금/방어 슬롯이 구분된다. |
| 1~6 역할별 비중이 갱신됐는가? | 전주 대비 초과/부족/변화가 기록된다. |
| 상태가 바뀐 종목이 분리됐는가? | 신규, 비중증가, 비중감소, 정리, 손실확대, 수익확대가 표시된다. |
| 변경 종목이 07 매수게이트를 통과했는가? | `G5/R1/G2/리스크` 중 하나로 판정된다. |
| 실행판 문장이 행동으로 이어지는가? | 추가매수, 정리/감축, 추가매수 금지, 수익 보호, 리스크 체크가 명확하다. |

오류 유형:

| 오류 | 의미 | 보정 |
|------|------|------|
| 스냅샷 반영 누락 | 사용자가 보유 현황을 줬는데 실제 보유현황이 최신화되지 않았다. | 실제 보유현황을 먼저 고치고 실행판을 재판정 |
| 기준일 불일치 | 실제 보유현황, 실행판, 주간기록의 날짜가 다르다. | 세 문서의 스냅샷 기준일 통일 |
| 계좌 역할 누락 | 장기/코어와 스윙/전술 판단이 섞였다. | 실행판과 매수게이트에 대상 계좌를 표기 |
| 비중 계산 누락 | 종목별 수익률만 보고 역할별 비중을 보지 않았다. | 1~6 역할별 변화표 보강 |
| 보유 변화 미분리 | 신규/감축/정리/수익보호가 섞였다. | 주간기록 주요 종목 변화표 보강 |
| 실행판 과잉반영 | 변화 없는 보유 전체를 실행판에 반복했다. | 실행판은 G5/R1/리스크 중심으로 축소 |

## 3차 검증 절차

### 1차. 프로세스 검증

판단이 정해진 문서 흐름을 통과했는지 확인한다.

| 질문 | 통과 기준 |
|------|-----------|
| 칼럼 인입을 통과했는가? | 성격, 출처 등급, 반영 등급이 있다. |
| 종목 아카이브에 출처가 있는가? | 직접/반복/확장 구분이 있다. |
| 판단원칙 6축 분류가 있는가? | 관찰, 후보, 조건부 실행 후보 등 분류가 있다. |
| 매수게이트 판정이 있는가? | G0~G5 또는 R1 코드가 있다. |
| 실행판 문장이 명확한가? | 조건, 수량, 무효화, 다음 트리거가 있다. |
| 실제 보유현황과 연결됐는가? | 매수/정리 후 보유 상태가 반영됐다. |

판정:

| 판정 | 의미 |
|------|------|
| 통과 | 절차를 지켰다. 결과가 나빠도 규칙 수정은 신중히 한다. |
| 부분 통과 | 일부 문서 누락이 있다. 해당 문서만 보정한다. |
| 실패 | 핵심 문서 흐름을 건너뛰었다. 판단 신뢰도를 낮춘다. |

### 2차. 결과 검증

실제 시장 반응이 판단과 얼마나 맞았는지 확인한다.

| 대상 | 확인 내용 |
|------|-----------|
| 매수 | 매수 후 가격, 실적, 이벤트 반응이 가설과 맞았는가? |
| 보류 | 보류 후 급등했는가, 아니면 리스크 회피가 맞았는가? |
| 금지 | 금지 종목이 계속 위험했는가, 과보수였는가? |
| 정리/감축 | 정리 후 리스크가 줄었는가, 너무 빨리 줄였는가? |
| 관찰 | 관찰 트리거가 실제로 발생했는가? |

판정:

| 판정 | 의미 |
|------|------|
| 적중 | 판단 방향과 시장 반응이 대체로 맞았다. |
| 조건부 적중 | 방향은 맞았지만 수량, 타이밍, 표현이 불완전했다. |
| 빗나감 | 판단과 실제 반응이 다르다. 원인 분류가 필요하다. |
| 판정 보류 | 결과 확인 기간이 아직 부족하다. |

### 3차. 규칙 보정

오류가 반복될 때만 규칙을 바꾼다. 단일 결과 하나로 철학이나 판단원칙을 바꾸지 않는다.

| 오류 | 의미 | 보정 |
|------|------|------|
| 누락 | 문서에는 있었지만 후보로 못 올렸다. | 종목 아카이브 → 판단원칙 연결 강화 |
| 과보수 | 소액 정찰병 가능 구간을 계속 놓쳤다. | 매수게이트 G3 조건 완화 또는 트리거 명확화 |
| 승격 실패 | 조건부 실행 후보 또는 실행 후보를 실행판에 올리지 않았다. | 판단원칙 → 매수게이트 → 실행판규칙 연결 보정 |
| 추격 | 너무 늦게 따라샀다. | 가격 위치와 돌파 후 첫 눌림 기준 강화 |
| 과분산 | 후보가 너무 많아 실행이 흐려졌다. | 실행판 조건부 대기 후보 최대 3개 유지 |
| 포화 | 같은 테마만 계속 늘렸다. | 포트폴리오 효과 게이트 강화 |
| 물타기 | 손실 보유를 가설 없이 늘렸다. | 실행판 금지 문장 강화 |
| 스냅샷 반영 누락 | 포폴 주간 현황을 받았지만 실제 보유현황과 실행판이 같은 기준으로 맞지 않았다. | 실제 보유현황 → 매수게이트 → 실행판 → 주간기록 순서 재적용 |
| 근거 실종 | 근거가 근거 인덱스/주간기록에 연결되지 않았다. | 근거 인덱스 필수화 |
| 문장 모호 | 실행판 문장이 행동으로 번역되지 않았다. | 실행판규칙 금지 문장 보강 |

보정 판정:

| 판정 | 처리 |
|------|------|
| 규칙 유지 | 결과 변동은 정상 범위다. |
| 문장 보정 | 실행판 표현만 고친다. |
| 게이트 보정 | 매수게이트 조건을 고친다. |
| 판단 보정 | 판단원칙 분류를 고친다. |
| 인입 보정 | 칼럼 인입/하단 체크 규칙을 고친다. |

## 오판 방지

| 오판 | 방지 규칙 |
|------|-----------|
| 수익이 났으니 판단이 맞았다 | 과정 검증을 별도로 한다. |
| 손실이 났으니 판단이 틀렸다 | 가설 훼손과 정상 변동성을 분리한다. |
| 놓친 상승은 어쩔 수 없다 | 어느 문서 단계에서 막혔는지 추적한다. |
| 단일 사례로 규칙을 바꾼다 | 반복 오류일 때만 원칙을 수정한다. |
| 검증이 감상문이 된다 | 오류 유형, 보정 문서, 다음 확인일을 남긴다. |

## 정합성 검증

### 1차 검증: 실행 문서와의 경계

| 항목 | 판정 | 이유 |
|------|------|------|
| 매수 판단 침범 여부 | 통과 | 검증규칙은 새 매수 판단을 만들지 않고 기존 판단을 평가한다. |
| 실행판 침범 여부 | 통과 | 실행판 문장을 평가하되 오늘 버튼을 직접 확정하지 않는다. |
| 실제 보유현황 침범 여부 | 통과 | 계좌 상태를 평가 자료로 쓰지만 보유 기록 자체는 변경하지 않는다. |

### 2차 검증: 피드백 루프 정합성

| 항목 | 판정 | 이유 |
|------|------|------|
| 주간기록 연결 | 통과 | 판단 검증 표를 주간기록에 반영하도록 했다. |
| 근거 인덱스 연결 | 통과 | 보정 문서와 근거 위치를 남기도록 했다. |
| 오류 유형 분류 | 통과 | 누락, 과보수, 추격, 과분산, 포화 등 반복 오류를 분류했다. |

### 3차 검증: 자산제곱 정합성

| 항목 | 판정 | 이유 |
|------|------|------|
| 정보 축적보다 연결 | 통과 | 검증 결과를 문서 보정 위치와 연결한다. |
| 결과보다 과정 | 통과 | 수익/손실만으로 판단을 평가하지 않는다. |
| 소외/FOMO 교정 | 통과 | 놓친 상승과 추격 매수를 모두 검증한다. |
| 지속 개선 | 통과 | 반복 오류만 규칙 보정으로 승격한다. |
