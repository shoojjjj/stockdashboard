## 한 줄 정의

칼럼 인입/반영/분석 에이전트는 총괄 브레인이 새 칼럼 요청으로 분류한 뒤 가장 먼저 작동하는 실무 라우터다. 칼럼의 내용을 이벤트/시나리오 업데이트, 철학, 시나리오 사고, 종목 아카이브, 판단원칙, 실행 영향으로 분해해 각 문서로 보낸다.

## 왜 1번 에이전트인가

칼럼은 매주 2~3회 추가된다. 칼럼이 들어올 때마다 새 분석 문서를 만들면 결론은 흩어지고, 매매 판단은 더 불명확해진다. 따라서 총괄 브레인이 새 칼럼 요청으로 분류한 뒤, 새 칼럼은 반드시 이 에이전트의 인입 절차를 거친다.

운영 원칙:

1. 새 칼럼은 실무상 먼저 인입한다.
2. 인입 후 문서 성격을 분류한다.
3. 반영 위치를 결정한다.
4. 필요한 기존 문서만 업데이트한다.
5. 긴 근거는 근거 인덱스 또는 주간기록으로 보낸다.
6. 매수 여부는 매수게이트와 실행판으로 넘긴다.

[자산제곱 매수게이트](<../07_매수게이트/자산제곱_매수게이트.md>)는 실행판에 올릴 후보를 심사하고, 최종 버튼은 [이번주 실행판](<../../../1_브리핑/자산제곱_브리핑/00_운용판단/이번주_실행판.md>)에서만 확정한다. 이 칼럼 인입 문서는 실행 판단을 대신하지 않는다.

승격 원칙:

- 칼럼 인입은 `매수 금지`를 기본값으로 두지 않는다.
- 자산제곱이 직접/반복 언급한 종목이나 섹터가 원칙, 시나리오, 가격 조건을 함께 제시하면 `실행 영향형`으로 분류한다.
- `실행 영향형`은 종목 아카이브에만 보관하지 않고 판단원칙, 매수게이트, 실행판규칙으로 라우팅한다.
- 조건이 충족된 후보를 계속 `후보군`으로만 남기면 승격 실패로 검증한다.

출처 구분:

- `직접 언급`, `하단 뷰포인트`, `원문 숫자`는 `칼럼 원문`으로 기록한다.
- 반영 등급 `A/B/C/D`, 실행 영향형, 승격 실패는 칼럼 원문 표현이 아니라 `에이전트 운용규칙`이다.
- 칼럼 논리에서 파생한 종목은 `칼럼 종합` 또는 `C 확장`으로 기록하고, 직접 추천처럼 쓰지 않는다.

## 인입 절차

### 1단계. 메타데이터 확인

새 칼럼을 읽기 전에 아래 항목을 먼저 고정한다.

| 항목 | 확인 내용 |
|------|-----------|
| 제목 | 원문 제목을 그대로 기록한다. |
| 날짜 | 칼럼 작성일 또는 저장일을 구분해 기록한다. |
| 경로 | 실제 파일 경로를 기록한다. |
| 최신성 | 기존 최신 칼럼보다 뒤인지 확인한다. |
| 폴더 | 투자 전략, 시장 리포트, 시나리오, 기술적 분석 등 위치를 확인한다. |
| 하단 추가 내용 | 칼럼 하단의 뷰포인트, 링크, PDF, 업데이트 문단을 반드시 확인한다. |

하단 추가 내용은 별도 체크 항목이다. 자산제곱 칼럼은 하단 뷰포인트나 첨부 링크에 실행 힌트가 붙는 경우가 있으므로 본문만 읽고 끝내면 안 된다.

### 2단계. 칼럼 성격 분류

칼럼은 하나의 성격만 갖지 않을 수 있다. 복합형이면 주 성격과 보조 성격을 나눠 기록한다.

| 성격 | 의미 | 반영 위치 |
|------|------|-----------|
| 철학형 | 투자 태도, 사고 방식, 시장을 보는 근간이 바뀐다. | `03_철학` |
| 시나리오형 | 앞으로의 경로, 변수, 확률, 이벤트가 바뀐다. | `02_이벤트시나리오`, `04_시나리오` |
| 종목 인벤토리형 | 신규 종목, 섹터, ETF, 한국주가 언급된다. | `05_종목아카이브`, 종목_종합정리 |
| 판단 기준형 | 종목 선별, 재무, 가격, 비중, 리스크 기준이 바뀐다. | `06_판단원칙` |
| 이벤트형 | 미중회담, 실적, 금리, 정책 등 특정 날짜 변수가 있다. | `02_이벤트시나리오`, 다가올 이벤트 |
| 실행 영향형 | 이번 주 매수/홀딩/매도·감축/조건부 대기 판단에 영향이 있다. | 매수게이트, 이번주 실행판 |
| 근거 보강형 | 기존 결론을 바꾸지는 않지만 근거가 보강된다. | 근거 인덱스, 필요 시 주간기록 |

### 2-1단계. 반영 등급 판정

칼럼 성격을 분류한 뒤에는 반영 등급을 붙인다. 등급은 실행 우선순위를 정하기 위한 것이며, 매수 결론이 아니다.

| 등급 | 의미 | 반영 위치 |
|------|------|-----------|
| A | 이번 주 매매 판단을 바꾼다. | 이번주 실행판, 현재 운용 판단, 근거 인덱스 |
| B | 후보군 또는 우선순위를 바꾼다. | 이번주 실행판, 근거 인덱스, 필요 시 종목_종합정리 |
| C | 종목 인벤토리에만 추가한다. | 종목_종합정리, 근거 인덱스 |
| D | 보관만 한다. | 근거 인덱스 또는 주간기록 |

| 이번 주 실행 영향 | 매수게이트로 넘길 정도인지 |

## 반영 규칙

### 원칙 1. 새 문서를 먼저 만들지 않는다

새 칼럼이 들어오면 기본 행동은 신규 문서 생성이 아니라 기존 문서 업데이트다.

새 문서 생성 허용 조건:

1. 새로운 에이전트 역할이 필요하다.
2. 기존 문서의 범위를 넘어서는 장기 체계가 생겼다.
3. 여러 칼럼을 묶는 대형 연구 노트가 필요하다.
4. 사용자가 명시적으로 새 문서 작성을 요청했다.

그 외에는 기존 문서에 결론만 반영하고, 긴 근거는 근거 인덱스와 주간기록으로 보낸다.

### 원칙 2. 직접 언급과 확장 해석을 분리한다

자산제곱이 직접 언급한 종목과, 칼럼 논리에서 파생한 종목은 같은 등급이 아니다.

| 출처 등급 | 의미 | 처리 |
|-----------|------|------|
| `A 직접` | 지금 반영하는 최신 칼럼 본문이나 하단 뷰포인트에 종목명/티커가 직접 나온 경우 | 종목 아카이브에 출처와 원문 맥락을 기록한다. |
| `B 반복` | 최신 칼럼 직접 언급은 아니지만 여러 칼럼에서 반복 직접 언급된 누적 핵심 후보 | 반복 근거와 최신 칼럼과의 연결성을 함께 기록한다. |
| `C 확장` | 최신 칼럼 직접 언급은 아니고, 이전 종목 인벤토리나 같은 테마에서 논리적으로 연결한 후보 | 직접 언급으로 쓰지 않고 `확장 후보`로만 기록한다. |
| ETF 대체 | 개별주 선별이 어려울 때만 검토하는 보조 수단 | 개별주 분석을 대체하지 않고, 포트폴리오 중복과 구성 종목을 확인한다. |

### 원칙 3. 칼럼은 매수 명령이 아니다

칼럼에서 좋은 구조가 언급되어도 바로 매수하지 않는다. 아래 단계를 통과해야 한다.

1. 종목 아카이브에 근거가 쌓인다.
2. 판단원칙 6축을 통과한다.
3. 가격, 이벤트, 비중, 포트폴리오 충돌을 확인한다.
4. 매수게이트에서 오늘 또는 이번 주 실행 가능성을 판정한다.
5. 실행판에 실제 행동으로 기록한다.

반대로, 위 단계를 통과한 뒤에도 계속 `관찰`이나 `후보군`으로만 두면 안 된다. 자산제곱 칼럼 원문/종합 원칙에 맞고 진입 조건이 충족된 경우에는 매수게이트의 `G3/G4/G5` 경로로 승격한다.

`G3/G4/G5`는 자산제곱 칼럼 원문 용어가 아니라 실행 경로를 관리하기 위한 `에이전트 운용규칙`이다.

## 오역 방지

| 오역 | 올바른 처리 |
|------|-------------|
| 새 칼럼이 나왔다 = 새 문서를 만든다 | 먼저 기존 문서에 라우팅한다. |
| 종목이 언급됐다 = 매수 후보 확정 | 종목 아카이브에 보관한 뒤 판단원칙으로 넘긴다. |
| 섹터가 좋다 = 관련주를 모두 산다 | 직접 수혜, 품질, 가격, 이벤트를 나눠 본다. |
| ETF가 있다 = 개별주 분석이 필요 없다 | ETF는 선별 실패 위험을 줄이는 보조 수단이다. |
| 이벤트가 끝났다 = 바로 실행 | 이벤트 후 가격 반응과 시나리오 변화까지 확인한다. |
| 하단 링크/PDF는 부가 자료 | 하단 자료도 칼럼의 일부로 본다. |

## 정합성 검증

### 1차 검증: 역할 경계

| 검증 항목 | 판정 | 이유 |
|-----------|------|------|
| 철학 침범 여부 | 통과 | 철학을 새로 정의하지 않고 반영 위치만 결정한다. |
| 시나리오 침범 여부 | 통과 | 시나리오 확률을 직접 정하지 않고 업데이트 필요 여부만 판단한다. |
| 종목 판단 침범 여부 | 통과 | 종목을 매수 후보로 확정하지 않고 출처와 근거만 라우팅한다. |
| 매수 판단 침범 여부 | 통과 | 매수 여부는 매수게이트와 실행판으로 넘긴다. |

### 2차 검증: 문서 혼란 방지

| 검증 항목 | 판정 | 이유 |
|-----------|------|------|
| 새 문서 남발 방지 | 통과 | 신규 문서 생성 조건을 제한했다. |
| 기존 문서 연결 | 통과 | 철학, 시나리오, 종목 아카이브, 판단원칙, 현재 운용 판단, 이번주 실행판으로 반영 경로를 고정했다. |
| 하단 뷰포인트 누락 방지 | 통과 | 메타데이터 확인 단계에 별도 항목으로 넣었다. |
| 직접 언급/확장 해석 분리 | 통과 | 종목 처리 등급을 분리했다. |

### 3차 검증: 자산제곱 정합성

| 검증 항목 | 판정 | 이유 |
|-----------|------|------|
| 정보 축적보다 연결 | 통과 | 칼럼 내용을 단순 보관하지 않고 문서별 역할로 연결한다. |
