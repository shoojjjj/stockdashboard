## 한 줄 정의

매수게이트 에이전트는 `좋다`, `관심 있다`, `자산제곱 관점에 맞다`를 바로 매수로 번역하지 않고, 정합성, 포트폴리오 효과, 이벤트 상태, 가격 위치, 수량/무효화 기준을 통과한 후보만 실행판으로 넘기는 실행 전 심사 문서다.

출처 구분:

- 좋은 종목과 오늘 매수 분리, 분할매수, 가격 반응 확인, 무효화 기준은 `칼럼 종합`이다.
- `G0~G5/R1`, `5대 게이트`, 승격 심사, 실행판 이관은 `에이전트 운용규칙`이다.
- `1주`, `전체 자산 1~2%`, `-10%/-20%`, `+40%/+60%` 같은 현재 계좌용 수치는 `포트폴리오 입력`이다.
- `20~80%` 스윙 비중은 칼럼 원문 근거가 있으나, 현재 두 계좌에 어떻게 배정할지는 `포트폴리오 입력`으로 표시한다.

## 핵심 원칙

### 1. 매수게이트는 최종 실행판이 아니다

매수게이트는 후보를 `실행판에 올릴 수 있는가`만 판단한다. 최종 매매 버튼, 수량 확정, 오늘의 행동 문장은 [이번주 실행판](<../../../1_브리핑/자산제곱_브리핑/00_운용판단/이번주_실행판.md>)에서만 확정한다.

### 2. 매수 없음은 기회 없음이 아니다

매수게이트에서 `보류`가 나와도 후보를 버리는 것이 아니다. 보류 사유를 명확히 남겨 다음 트리거가 오면 다시 통과시킨다.

| 보류 사유 | 의미 | 다음 트리거 |
|-----------|------|-------------|
| 가격 미확인 | 구조는 맞지만 추격 구간이다. | 눌림, 재지지, 돌파 후 첫 확인 |
| 이벤트 전 | 실적, 회담, 정책 발표 전이다. | 이벤트 결과와 가격 반응 |
| 포트폴리오 포화 | 이미 같은 축이 많다. | 기존 축 정리, 비중 하락, 다른 축 빈칸 발생 |
| 무효화 부재 | 틀렸을 때 나갈 기준이 없다. | 가격 기준 또는 가설 훼손 기준 설정 |
| 정보 부족 | 실적, 현금흐름, 선행 변수 확인이 부족하다. | 추가 데이터 확인 |

### 2-1. 상승장에서는 참여 후보를 먼저 찾는다

상승 시나리오가 살아 있고 자산제곱 언급 흐름이 가격으로 확인되면, 매수게이트는 `살 수 없는 이유`만 찾지 않는다. 먼저 `작게 참여할 수 있는 구조`가 있는지 확인한다.

| 상황 | 처리 |
|------|------|
| 상승 흐름 유지 + 자산제곱 직접/반복 언급 + 무효화 기준 있음 | `G3 조건부 정찰병` 우선 검토 |
| 장기 구조와 가격 재지지가 동시에 확인됨 | `G4 1차 매수 가능` 검토 |
| 불확실하지만 틀렸을 때 기준이 명확함 | 금지보다 수량 축소 |
| 조건이 맞는데도 계속 매수 없음 | 과보수 오류 |
| 추격 급등, 무효화 부재, 포트폴리오 포화 | 매수 금지 또는 `G2 보류` |

### 3. 과보수와 FOMO를 동시에 막는다

자산제곱식 매수는 `아무것도 안 함`도 아니고 `좋아 보이면 바로 삼`도 아니다.

| 상황 | 올바른 처리 |
|------|-------------|
| 장기 구조는 맞지만 단기 가격이 앞섬 | 관찰 또는 조건부 후보 |
| 가격은 눌렸지만 구조가 훼손됨 | 금지 |
| 정합성, 포트폴리오 효과, 무효화 기준이 명확함 | 소액 정찰병 가능 |
| 실적 직전이고 변동성 큼 | 대형 진입 금지, 이벤트 후 재판정 |
| FOMO가 강함 | 1주/소액 정찰병만 검토, 무효화 기준 없으면 금지 |

### 4. 조건 충족 시 실행판 승격을 기본값으로 둔다

매수게이트는 매수를 막기 위한 문서가 아니라, 실행 가능한 후보와 아직 아닌 후보를 분리하는 문서다. 판단원칙을 통과한 후보가 아래 조건을 충족하면 `보류`가 아니라 `G3/G4/G5`로 판정한다.

승격 공식(`에이전트 운용규칙`):

```text
자산제곱 직접/반복 언급
→ 판단원칙 6축 통과 또는 대부분 통과
→ 가격·이벤트·포트폴리오·무효화 조건 충족
→ 실행판 이관
```

| 상황 | 게이트 판정 |
|------|-------------|
| 가격 구간만 오면 살 수 있고 수량/무효화가 있다 | `G3 조건부 정찰병` |
| 가격, 이벤트, 비중, 무효화가 모두 맞고 1차 진입 가능 | `G4 1차 매수 가능` |
| 기존 보유의 가설과 가격이 유지되고 추가 비중이 합리적 | `G5 추가매수 가능` |
| 조건이 충족됐는데도 계속 보류 | 과보수 오류 |

`추격 금지`는 나쁜 가격에서 사지 말라는 뜻이다. 자산제곱이 제시한 눌림, 재지지, 분할매수 구간에 들어왔고 무효화 기준이 있으면 실행판으로 올린다.

저점 놓침 방지 규칙:

- 모든 조건이 완벽해질 때까지 기다리지 않는다. 완벽 확인은 대개 가격 상승 뒤에 온다.
- 정합성, 포트폴리오 역할, 무효화 기준이 있고 가격이 저점권/첫 재지지/첫 눌림에 있으면 `G3 조건부 정찰병`을 허용한다.
- 5대 게이트 중 4개 이상이 충족되고 대형 이벤트 리스크가 지나갔으면 `G4 1차 매수 가능`으로 승격한다.
- 불확실성은 `매수 금지`가 아니라 `수량 축소`로 처리한다. 단, 무효화 기준이 없으면 어떤 경우에도 매수하지 않는다.
- 이미 저점이 지나갔다면 복수 조건 확인을 이유로 영구 보류하지 말고, 첫 눌림/재지지/돌파선 방어를 새 진입 기준으로 다시 만든다.

`G3/G4/G5`는 자산제곱 원문 용어가 아니라 칼럼 원칙을 실행판으로 넘기기 위한 코드다.

## 입력 조건

매수게이트는 모든 종목을 처음부터 분석하지 않는다. 아래 입력 출처 중 하나가 있어야 하며, 실행판 이관 전에는 반드시 [판단원칙](<../06_판단원칙/자산제곱_판단원칙.md>) 분류가 있어야 한다.

예외적으로 사용자가 특정 종목을 즉시 물어본 경우에는 `예비 심사`를 할 수 있다. 그러나 판단원칙 분류가 없으면 `G3 조건부 정찰병` 이상으로 승격할 수 없다.

| 입력 출처 | 조건 |
|-----------|------|
| 칼럼 인입 | 반영 등급 `A` 또는 `B`, 혹은 출처 등급 `A 직접`. 단, 판단원칙 분류가 없으면 예비 심사까지만 가능 |
| 종목 아카이브 | 직접 언급 또는 반복 근거가 있고 출처가 확인됨 |
| 판단원칙 | `조건부 실행 후보`, `실행 후보`, `보유 재검토` 중 하나 |
| 이번주 실행판 | 기존 후보의 조건 재확인 필요 |
| 실제 보유현황 | 추가매수, 정리, 손절, 수익 보호 판단 필요 |
| 포폴 주간 스냅샷 | 계좌 전체 비중 변화와 역할별 쏠림을 `G5/R1/G2/리스크`로 재분류할 때 필요 |
| 사용자 요청 | 특정 종목 매수 가능 여부 질문. 단, 판단원칙 분류가 없으면 `G1 관찰` 또는 `G2 보류`까지만 가능 |

입력 조건을 충족하지 못하면 매수게이트로 오지 않고 `관찰` 또는 `종목_종합정리 보관`으로 둔다. 판단원칙 분류가 없는 후보는 실행판으로 이관하지 않는다.

## 승격 심사 발생 시점

매수게이트는 상시 자동 감시 에이전트가 아니다. 아래 입력이 들어오거나 사용자가 에이전트 실행을 요청했을 때 후보를 재심사하고, 조건이 충족되면 `G3/G4/G5`로 승격한다.

| 시점 | 심사 대상 | 승격 가능 코드 |
|------|-----------|----------------|
| 신규 칼럼 인입 후 | 자산제곱이 직접 언급한 종목, 섹터, 자산군 | `G1`~`G4` |
| Clippings 처리 완료 후 | 새 칼럼에서 나온 조건부 후보와 기존 후보의 변화 | `G2`~`G4` |
| 주간 보유 스냅샷 입력 후 | 실제 보유 종목의 추가매수, 물타기, 수익 보호, 정리 필요 여부 | `G5`, `R1`, `리스크` |
| 이벤트 종료 후 | 실적, Computex, FOMC, 미중/중동 이벤트 이후 가격 반응 | `G3`, `G4`, `G2` |
| 가격 조건 도달 확인 후 | 저점권, 첫 눌림, 돌파선 재지지, 지정 지지선 방어 | `G3`, `G4` |
| 사용자가 특정 종목을 물었을 때 | 해당 종목의 예비 심사 또는 기존 후보 재심사 | `G1`~`G4` |

승격은 `언급` 시점이 아니라 `언급 + 조건 확인` 시점에 일어난다. 조건부 후보가 조건을 충족했는데도 다음 실행판까지 `관찰`에 남아 있으면 과보수 오류로 본다.

## 5대 게이트

매수게이트는 아래 5개를 모두 확인한다. 이 기준은 [판단원칙](<../06_판단원칙/자산제곱_판단원칙.md>)의 후보 분류를 실행 전 심사표로 바꾼 `에이전트 운용규칙`이다.

| 게이트 | 질문 | 통과 기준 |
|--------|------|-----------|
| 1. 정합성 | 최신 칼럼 또는 반복 원칙과 맞는가? | 구조적 축, 출처 등급, 판단원칙이 연결된다. |
| 2. 포트폴리오 효과 | 부족한 축을 채우는가, 쏠림을 키우는가? | 빈칸 보강 또는 위험 분산에 기여한다. |
| 3. 이벤트 상태 | 실적, 정책, 회담, 금리 이벤트가 확인됐는가? | 이벤트 통과 또는 이벤트 전 소액 허용 사유가 있다. |
| 4. 가격 위치 | 추격이 아니라 눌림, 재지지, 돌파 후 확인인가? | 손절/재검토 좌표를 둘 수 있다. |
| 5. 수량/무효화 | 1주 또는 소액으로 설명 가능하고 틀렸을 때 기준이 있는가? | 수량, 무효화 가격, 가설 훼손 기준이 있다. |

판정:

| 통과 수 | 판정 | 다음 행동 |
|---------|------|-----------|
| 5개 통과 | 실행판 매수 후보 | 이번주 실행판으로 이관 |
| 4개 통과, 가격만 미확인 | 조건부 대기 후보 | 가격 트리거를 붙여 실행판 또는 근거 기록 |
| 정합성은 있으나 이벤트/가격 미확인 | 관찰 | 이벤트 보드 또는 근거 인덱스 |
| 포트폴리오 쏠림 또는 무효화 기준 부재 | 보류 | 조건 보강 전 실행 금지 |
| 급등 추격, 실적 직전 대형 진입, FOMO | 금지 | 실행판의 해당 블록 금지 조건으로 이관 |

## 포폴 주간 스냅샷 게이트

사용자가 이번 주 보유 종목 현황을 전달하면 매수게이트는 신규 종목 발굴기가 아니라 `기존 계좌의 행동 판정기`로 작동한다.

필수 입력:

| 항목 | 기준 |
|------|------|
| 스냅샷 기준일 | 언제의 계좌인지 명확해야 한다. |
| 총자산/예수금 | 현금성 여력과 위험자산 비중 계산에 필요하다. |
| 대상 계좌 | 장기/코어, 스윙/전술, 현금/방어 중 어디의 판단인지 명확해야 한다. |
| 종목/수량/평가금/수익률 | 실제 보유현황에 반영할 최소 단위다. |
| 전주 대비 변화 | 신규, 추가, 감축, 정리, 손익 변화가 있으면 표시한다. |
| 역할 번호 | 1~6 바구니 중 어디에 속하는지 분류한다. |

판정 순서:

1. 전체 계좌를 먼저 본다. 현금성, 위험자산, 1~6 역할별 쏠림을 계산한다.
2. 대상 계좌를 확정한다. `6212-7075`는 장기/코어, `6555-6037`은 스윙/전술로 본다.
3. 상태가 바뀐 종목만 게이트에 올린다. 변화 없는 보유 전체를 다시 심사하지 않는다.
4. 기존 보유 추가매수는 `G5 추가매수 가능`으로만 표현한다.
5. 가설 훼손, 역할 중복, 비중 과다, 손실 기준 침범은 `R1 정리/감축 검토`로 표현한다.
6. 가설은 살아 있지만 가격, 이벤트, 비중이 애매하면 `G2 보류` 또는 `추가매수 금지`로 둔다.

계좌 역할 게이트:

| 계좌 역할 | G5 가능 조건 | R1/금지 조건 |
|-----------|--------------|--------------|
| 장기/코어 | 성장 스토리와 장기 구조가 유지되고, 과도한 쏠림이 아니다. | 장기 가설 훼손, 단일 테마 과밀, 스윙 손실 보전 목적 매도 |
| 스윙/전술 | 시나리오, 가격 재지지, 이벤트, 무효화 기준이 모두 맞고 20~80% 범위 안이다. | 손실 종목을 장기화하려는 판단, 무효화 부재, 신고가 추격 |
| 현금/방어 | 방어층을 줄여도 더 높은 우선순위의 조건부 후보가 있다. | 예수금 부족, 방어층 훼손, 장기채를 예수금처럼 쓰는 판단 |

보유 종목 판정표:

| 상태 | 게이트 | 실행판 처리 |
|------|--------|-------------|
| 가설 유지, 가격 재지지, 비중 여유 있음 | `G5 추가매수 가능` | 추가매수 가능 |
| 가설 유지, 가격/이벤트 미확인 | `G2 보류` | 보유, 추가매수 금지 |
| 수익 크고 비중 과다 | `리스크` | 수익 보호 |
| 가설 훼손 또는 역할 충돌 | `R1 정리/감축 검토` | 정리/감축 검토 |
| 손실 확대이나 가설 보강 없음 | `G2 보류` | 물타기 금지 |

제한:

- 스냅샷 기준일이 불명확하면 `G5`를 부여하지 않는다.
- 현재가, 이벤트, 비중, 무효화 중 하나가 비면 `추가매수 가능`으로 쓰지 않는다.
- 정리는 `매도 확정`이 아니라 `R1 정리/감축 검토`로 먼저 실행판에 올린다.
- 보유 종목은 신규 후보보다 근거가 더 엄격해야 한다.

## 세부 게이트

### 0단계. 후보 자격 확인

아래 중 하나라도 없으면 정식 게이트 심사에 올리지 않는다.

- 출처 등급: `A 직접`, `B 반복`, `C 확장` 중 하나
- 판단원칙 분류: `조건부 실행 후보`, `실행 후보`, `보유 재검토` 중 하나
- 근거 링크: 칼럼, 종목 아카이브, 근거 인덱스 중 하나
- 시간축: 단기, 중기, 장기, 혼합 중 하나

판단원칙 분류가 없는 경우:

- `G3 조건부 정찰병`, `G4 1차 매수 가능`, `G5 추가매수 가능` 금지
- 필요하면 먼저 판단원칙 6축 점검으로 되돌린다.
- 단순 가격 질문이면 `예비 보류`만 남긴다.

### 1단계. 정합성

질문:

- 최신 칼럼의 구조적 축과 직접 연결되는가?
- 반복 언급된 자산제곱 칼럼 원문/종합 원칙과 맞는가?
- 단순 테마주인가, 병목/현금흐름/정책/수급 중 하나가 설명되는가?
- 한국주, ETF, 코인, 원자재라면 미국장 후보와 섞지 않고 별도 축으로 분류했는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 직접 언급 또는 반복 원칙과 연결되고, 구조적 축이 명확하다. |
| 조건부 | 확장 후보지만 현재 운용 판단과 연결된다. |
| 탈락 | 단순 관련주, 막연한 테마, 출처 불명확 후보다. |

### 2단계. 포트폴리오 효과

질문:

- 기존 보유와 같은 방향으로만 움직이는가?
- 전력/AI/반도체/원자재/금/코인/방산/에너지 중 어느 바구니인가?
- 현재 포트폴리오의 빈칸을 채우는가?
- 기존 손실 포지션 물타기인가, 새 가설에 따른 분할 진입인가?
- 단기채는 현금성으로 분류했는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 부족한 축을 채우거나 리스크 분산에 기여한다. |
| 조건부 | 좋은 후보지만 일부 중복된다. 수량을 줄이면 가능하다. |
| 탈락 | 이미 포화된 테마를 더 키운다. |

### 3단계. 이벤트 상태

질문:

- 실적 발표 전인가, 후인가?
- 실적은 절대적으로 좋았는가, 기대치 대비 좋았는가?
- 발표 후 주가 반응은 강했는가, 기대 이하로 빠졌는가?
- 미중회담, FOMC, CPI, 정책 발표, 지정학 이벤트 전후인가?
- 이벤트 후 가격이 지지되는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 이벤트가 지나갔고 가격 반응이 확인됐다. |
| 조건부 | 이벤트 전이지만 수량이 작고 무효화 기준이 명확하다. |
| 탈락 | 실적 직전 대형 진입, 이벤트 결과 미확인, 뉴스 추격이다. |

### 4단계. 가격 위치

질문:

- 신고가 추격인가, 눌림인가?
- 컵앤핸들, 횡보 상단 돌파, 추세 반전, 재지지 중 하나가 있는가?
- 돌파 후 첫 눌림인가, 이미 멀리 달린 뒤인가?
- 손절 또는 재검토 좌표가 있는가?
- 좋은 실적에도 빠진 경우, 기대치 문제인지 구조 훼손인지 구분했는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 눌림, 재지지, 돌파 후 첫 확인, 추세 반전 중 하나다. |
| 조건부 | 구조는 맞지만 가격이 아직 앞서 있다. |
| 탈락 | 급등 추격, 무효화 좌표 부재, 변동성 확대 구간이다. |

### 5단계. 수량과 무효화

질문:

- 1주 또는 전체 자산의 1~2% 정찰병으로 설명 가능한가?
- 고가 종목이면 1주도 과도한 비중이 아닌가?
- 무효화 기준은 가격 기준과 가설 기준으로 나뉘어 있는가?
- 추가매수 조건과 실패 조건이 둘 다 있는가?
- 매수 후 주간기록에서 사후검증할 수 있는가?

판정:

| 판정 | 기준 |
|------|------|
| 통과 | 수량, 무효화, 사후검증 기준이 모두 있다. |
| 조건부 | 수량은 작지만 무효화 기준 보강이 필요하다. |
| 탈락 | 틀렸을 때 무엇을 할지 정해져 있지 않다. |

## 판정 코드

매수게이트 출력은 아래 코드 중 하나만 쓴다. 이 코드는 칼럼 원문이 아니라 에이전트 내부 실행 코드다.

| 코드 | 의미 | 실행판 처리 |
|------|------|-------------|
| `G0 제외` | 게이트 자격 없음 | 실행판에 올리지 않는다. |
| `G1 관찰` | 구조는 있으나 조건 부족 | 원칙적으로 실행판 제외. 트리거가 명확할 때만 조건부 대기 |
| `G2 보류` | 정합성은 있으나 가격/이벤트/포트폴리오 문제 | 조건부 대기 또는 해당 블록의 금지 조건 |
| `G3 조건부 정찰병` | 특정 조건 충족 시 1주/소액 가능 | 조건과 무효화 기준을 붙여 실행판 이관 |
| `G4 1차 매수 가능` | 5대 게이트 통과 | 실행판 매수 후보 |
| `G5 추가매수 가능` | 기존 보유의 가설과 가격이 모두 유지 | 실행판 추가매수 후보 |
| `R1 정리/감축 검토` | 보유 종목의 가설 또는 비중 문제 | 실행판 정리/감축 후보 |

## 유형별 적용

### 신규 미국 개별주

필수 조건:

- 출처 등급과 반영 등급이 있다.
- 판단원칙 6축 중 가격 좌표와 무효화 기준이 포함되어 있다.
- 기존 보유와 같은 축이면 왜 더 사야 하는지 설명된다.
- 실적 전 대형 진입은 금지한다.

### 기존 보유 추가매수

필수 조건:

- 기존 매수 가설이 살아 있다.
- 손실 보유라면 물타기가 아니라 가설 강화인지 확인한다.
- 수익 보유라면 추격이 아니라 분할 확대인지 확인한다.
- 같은 테마 비중이 과도하면 추가매수보다 수익 보호가 우선이다.

### ETF

필수 조건:

- ETF는 전략이 아니라 매수수단으로만 본다.
- 구성 종목이 현재 보유와 과도하게 겹치지 않아야 한다.
- 개별주 선별 실패 위험을 줄이는 목적이어야 한다.
- 섹터 전체 노출이 필요한 축인지 확인한다.

### 한국주

필수 조건:

- 미국장 후보와 섞지 않는다.
- 환율, 외국인 수급, 개인 신용, 신고가/눌림 위치를 별도 확인한다.
- 글로벌 사이클 수혜축이라도 고점 추격은 금지한다.
- 한국장은 변동성이 크므로 미국장 정찰병 규칙을 그대로 복사하지 않는다.

### 코인/원자재

필수 조건:

- 포트폴리오 내 역할이 명확해야 한다. 성장 베팅, 보험, 유동성 베팅을 섞지 않는다.
- 비트코인/코인 관련주는 정책, 유동성, 리스크 온 신호를 함께 본다.
- 금/원자재는 보험인지 주도 베팅인지 먼저 정한다.
- 기존 보유와 중복이면 추가매수보다 역할 재정의가 우선이다.

## 실행판 이관 문장 규칙

실행판으로 넘길 때는 긴 분석을 쓰지 않는다. 세부 배치와 문장 구조는 [실행판규칙](<../08_실행판규칙/자산제곱_실행판규칙.md>)을 따른다. 매수게이트에서는 아래처럼 버튼 중심 문장만 넘긴다.

| 게이트 판정 | 실행판 문장 |
|-------------|-------------|
| `G3 조건부 정찰병` | `조건 충족 시 1주/소액 정찰병. 무효화: ...` |

## 오판 방지

| 오판 | 방지 규칙 |
|------|-----------|
| 매수 후보가 많으면 많이 산다 | 실행판 후보는 최대 1~3개만 남긴다. |
| 매수 없음은 기회 없음이다 | 보류 사유와 다음 트리거를 남긴다. |
| 하락했으니 싸다 | 구조 유지와 무효화 기준이 없으면 금지다. |
| 실적이 좋으면 산다 | 기대치 대비 반응과 가격 지지를 확인한다. |
| 1주면 항상 안전하다 | 고가 종목은 1주도 큰 비중일 수 있다. |
| ETF면 안전하다 | 구성 종목 중복과 테마 포화를 확인한다. |
| 기존 보유 손실은 물타기로 해결한다 | 가설 강화가 없으면 추가매수 금지다. |
| 칼럼 직접 언급은 매수 신호다 | 직접 언급은 게이트 자격일 뿐 매수 결론이 아니다. |

## 정합성 검증

### 1차 검증: 판단원칙과의 경계

| 항목 | 판정 | 이유 |
|------|------|------|
| 6축 판단 반복 여부 | 통과 | 판단원칙의 6축을 다시 평가하지 않고 실행 전 5대 게이트로 압축한다. |
| 좋은 후보와 실행 후보 분리 | 통과 | 판단원칙 통과 후에도 가격, 이벤트, 수량을 다시 확인한다. |
| 종목 아카이브 침범 여부 | 통과 | 출처와 근거 보관은 종목 아카이브에 둔다. |

### 2차 검증: 실행판과의 경계

| 항목 | 판정 | 이유 |
|------|------|------|
| 최종 매매 버튼 침범 여부 | 통과 | 최종 버튼은 이번주 실행판에서만 확정한다. |
| 수량 확정 침범 여부 | 통과 | 이 문서는 수량 범위와 조건만 제시하고 체결 기록은 하지 않는다. |
| 긴 분석 방지 | 통과 | 실행판 이관 문장은 짧은 버튼 문장으로 제한한다. |

### 3차 검증: 자산제곱 정합성

| 항목 | 판정 | 이유 |
|------|------|------|
| 장기 구조와 단기 매수 분리 | 통과 | 장기 정합성이 있어도 가격/이벤트가 없으면 보류한다. |
| 전술 슬롯 반영 | 통과 | 조건이 일부 충족되면 소액 정찰병을 허용해 과보수를 줄인다. |
| FOMO 방지 | 통과 | 급등 추격, 실적 직전 대형 진입, 무효화 부재를 금지한다. |
| 사후검증 가능성 | 통과 | 매수 전부터 무효화와 주간기록 검증 기준을 요구한다. |
