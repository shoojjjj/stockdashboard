# 자산제곱 이벤트/시나리오 업데이트

> 목적: 새 칼럼과 텔레그램 코멘트에서 나온 이벤트, 변수, 시나리오 변화를 `살아있는 관리 대상`으로 등록하고 종료 조건까지 추적한다.

## 한 줄 정의

이벤트/시나리오 업데이트 에이전트는 칼럼에서 나온 `다가올 변수`를 등록하고, 이벤트 전후로 어떤 시나리오가 강화·약화·무효화되는지 추적하는 문서다.

출처 구분:

- 이벤트, 날짜, 종목, 직접 조건이 칼럼에 있으면 `칼럼 원문`으로 기록한다.
- 발표/이벤트 후 가격 반응을 확인한다는 원칙은 여러 칼럼에 반복되는 `칼럼 종합`이다.
- 이벤트 확인 기준을 고정 기간으로 만들지 않는다. 최종 기준은 결과 이후 가격, 섹터, 수급, 시나리오 변화다.

## 왜 02번인가

총괄 브레인이 새 칼럼 요청으로 분류한 뒤, 새 칼럼은 실무상 먼저 [칼럼 인입/반영/분석](<../01_칼럼인입/자산제곱_칼럼인입_반영_분석.md>)을 통과한다. 그 다음 즉시 확인해야 할 것은 `철학`이 아니라 `다가오는 이벤트와 변수`다.

이유:

1. 자산제곱 칼럼은 특정 이벤트 전후의 대응을 자주 다룬다.
2. 이벤트가 끝났는데도 문서에 남아 있으면 판단이 흐려진다.
3. 이벤트가 계속 추가될수록 `이번 주 매매`와 `장기 원칙`이 뒤섞인다.
4. 이벤트를 먼저 분리해야 철학, 시나리오, 종목 판단을 과잉 수정하지 않는다.

따라서 02번은 `이벤트와 단기 변수의 관문`이고, 04번 시나리오 사고는 `복수 미래를 설계하는 장기 프레임워크`다.

## 이벤트 생애주기

이벤트는 등록만 하면 안 된다. 반드시 종료 조건까지 둔다.

| 단계 | 질문 | 반영 위치 |
|------|------|-----------|
| 1. 등록 | 어떤 이벤트가 언제 있는가? | 다가올 이벤트 |
| 2. 사전 판단 | 이벤트 전에는 매수, 홀딩, 매도·감축, 조건부 대기 중 무엇인가? | 이번주 실행판 |

| 4. 결과 확인 | 실제 결과와 가격 반응은 어떤가? | 이벤트 문서, 근거 인덱스 |
| 5. 시나리오 업데이트 | 기존 시나리오가 강화, 약화, 무효화됐는가? | 04 시나리오 사고 |
| 6. 실행 이관 | 이번 주 살 수 있는 후보가 생겼는가? | 07 매수게이트 |
| 7. 종료 | 더 이상 판단에 영향을 주지 않는가? | 시나리오 변경기록/근거 인덱스/주간기록에 요약 후 현재 이벤트 보드에서 제거 |

## 판단 규칙

### 1. 이벤트는 매수 핑계가 아니다

이벤트가 있다는 이유만으로 매수하지 않는다. 이벤트는 `불확실성`이다. 매수는 이벤트가 끝난 뒤 가격 반응, 실적 반응, 수급 반응이 확인될 때만 가능하다.

예외:

- 구조 변화가 명확하고 가격이 아직 반영하지 않은 경우
- 이미 정찰병으로만 접근하는 경우
- 손실 허용 범위가 작고 사후 검증 조건이 명확한 경우

### 2. 이벤트 전과 이벤트 후를 분리한다

같은 종목이라도 이벤트 전 판단과 이벤트 후 판단은 다르다.

| 구분 | 원칙 |
|------|------|
| 이벤트 전 | 추격 매수 금지, 비중 확대 제한, 조건부 정찰병만 허용 |
| 이벤트 직후 | 숫자보다 가격 반응과 가이던스를 먼저 확인 |
| 이벤트 후 가격 반응 | 눌림 방어, 갭 메우기, 거래량, 섹터 반응 확인. 최종 기준은 가격과 시나리오 변화다. |
| 이벤트 종료 | 실행판에 남길지, 이벤트 문서에서 제거할지 결정 |

### 3. 결과보다 반응이 중요하다

실적이 좋아도 주가가 하락하면 기대치가 너무 높았을 수 있다. 뉴스가 나빠도 가격이 버티면 악재 소화일 수 있다.

확인 순서:

1. 이벤트 결과가 예상 대비 좋은가
2. 가격이 그 결과를 어떻게 받아들였는가
3. 섹터와 동종 기업도 같은 반응인가
4. 기존 시나리오가 강화됐는가 약화됐는가
5. 이번 주 실행할 수 있는 가격 조건이 생겼는가

실행 이관 원칙:

- 이벤트 전에는 불확실성을 관리한다.
- 이벤트 후에는 결과와 가격 반응을 확인한다. 확인 기간을 고정하지 않고, 이벤트 성격과 가격·섹터·수급 반응으로 판단한다.
- 긍정 조건과 가격 반응이 맞고, 관련 종목이 판단원칙/매수게이트 조건을 충족하면 실행판으로 보낸다.
- 이벤트가 끝났는데도 계속 `확인 전 보류`로 남기면 승격 실패 또는 이벤트 방치 오류로 검증한다.

### 4. 이벤트 종료를 명시한다

이벤트가 종료됐는데 계속 살아 있으면 매번 “다음 이벤트”를 핑계로 판단이 미뤄진다.

종료 기준:

- 예정 결과가 발표됐다.
- 시장 가격 반응과 섹터 반응이 확인됐다.
- 매수게이트로 넘길 후보가 있거나 없다는 결론이 났다.
- 남은 내용이 장기 구조라면 04 시나리오 또는 06 판단원칙으로 이관했다.
- 단기 이벤트 문구는 시나리오 변경기록, 근거 인덱스, 주간기록에 요약한 뒤 다가올 이벤트 문서에서 제거했다.

## 오역 방지

| 오역 | 올바른 처리 |
|------|-------------|
| 이벤트가 있다 = 매수를 기다린다 | 이벤트별 종료 조건을 두고 끝나면 판단한다. |
| 이벤트가 끝났다 = 바로 매수한다 | 결과와 가격 반응을 확인한 뒤 매수게이트로 보낸다. |
| 이벤트가 많다 = 문서를 계속 만든다 | 다가올 이벤트 문서에 등록하고 종료 후 시나리오 변경기록/근거 인덱스/주간기록에 요약한 뒤 현재 보드에서 제거한다. |
| 시나리오 변화 = 철학 변화 | 단기 변수는 02번, 장기 사고 체계는 04번으로 분리한다. |
| 좋은 결과 = 좋은 매수 | 기대치와 가격 반응을 확인해야 한다. |
| 악재 = 무조건 매도 | 가격이 버티는지, 가설이 깨졌는지 분리한다. |
