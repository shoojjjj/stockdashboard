# 오늘 매매 회의 운용

> 목적: `오늘 매매할 것은?` 요청이 들어왔을 때 네 에이전트의 입력을 순서대로 모아 최종 확인 문장을 만든다.

## 입력 최신성 확인

회의 시작 전에 아래 기준을 확인한다.

| 입력 | 확인할 문서 |
|------|-------------|
| 자산제곱 최신 결론 | `1_브리핑/자산제곱_브리핑/00_운용판단/이번주_실행판.md` |
| 자산제곱 운용 원칙 | `1_브리핑/자산제곱_브리핑/00_운용판단/현재_운용_판단.md` |
| 소수몽키 일일 신호 | `1_브리핑/소수몽키_브리핑/일일보고/` 최신 문서 |
| 소수몽키 신호판 | `0_주식_에이전트/소수몽키_에이전트/03_신호_태그화/신호판/` 최신 문서 |
| 실제 보유와 여력 | `1_브리핑/포트폴리오_브리핑/00_현황/실제_보유현황.md` |

날짜가 맞지 않으면 그 입력은 `구버전`으로 표시하고 결론 강도를 낮춘다.

## 진행 순서

1. 자산제곱이 먼저 말한다.
   - 오늘 실행 가능한 후보가 있는가?
   - 게이트는 `G0~G5/R1` 중 어디인가?
   - 무효화 조건과 이벤트 조건은 무엇인가?
2. 소수몽키가 의견을 낸다.
   - 같은 종목 또는 같은 테마에 `SM-S2~SM-S3` 신호가 있는가?
   - 단기 모멘텀은 우호적인가, 과열인가?
   - 이벤트 촉매가 오늘 또는 이번 주에 있는가?
3. 포트폴리오가 가세한다.
   - 이미 들고 있는가?
   - 같은 테마가 과밀인가?
   - 현금/방어 여력이 있는가?
   - 고수익 보호 또는 손실 정리가 먼저인가?
4. 중재자가 합의와 충돌을 분리한다.
   - 세 에이전트가 같은 방향이면 사용자 확인 후보로 올린다.
   - 한 에이전트라도 강하게 반대하면 조건부 대기 또는 보류로 낮춘다.
   - 포트폴리오가 막으면 좋은 종목이어도 `추가 금지`를 유지한다.

## 회의 테이블

| 후보 | 자산제곱 | 소수몽키 | 포트폴리오 | 충돌 | 중재 결론 |
|------|----------|----------|------------|------|-----------|
| 티커/테마 | G코드/판단 | SM-S코드/신호 | 여력/중복/보유 | 있음/없음 | 매수 후보/조건부 대기/추가 금지/수익 보호/정리 검토 |

## 사용자에게 보여줄 결론 형식

```text
기준시각:
입력 최신성:

오늘 바로 할 것:
오늘 하지 말 것:
조건부 대기:
보유/수익 보호:
정리/감축 검토:

핵심 합의:
핵심 불합의:
사용자 확인 필요:
```

## 금지

- 중재자가 새 종목을 추가하지 않는다.
- 소수몽키 신호만으로 바로 매수 결론을 내리지 않는다.
- 자산제곱 원칙만으로 포트폴리오 여력을 있다고 가정하지 않는다.
- 보유 중이라는 이유만으로 자동 추가매수하지 않는다.
- 사용자의 확인 없이 최종 주문 문장으로 쓰지 않는다.
