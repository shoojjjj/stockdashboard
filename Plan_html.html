✅ 한 줄 결론

가능하다. 다만 "대시보드 한 창에서 Python 로봇까지 전부 Vercel 안에서 돌린다"는 방식은 현실적으로 어렵고, 보기용 대시보드(Vercel) + 자동 수집·분석(별도 실행)으로 나누는 것이 정답에 가깝다.
역할어디서 돌릴까
매일 아침 볼 화면Vercel (웹 대시보드)
텔레그램 수집, OCR, 신호판 생성PC 스케줄러 또는 GitHub Actions
결과물 저장JSON / MD → Git 또는 Supabase
AI 회의 (토론 중재)Cursor에서 질문, 또는 API 연동

Vercel은 "매일 열어보는 창"에 최적이고, Telethon·OCR 같은 무거운 작업은 밖에서 돌려서 결과만 올리는 구조가 안정적이다.

📦 지금 아카이브가 이미 갖춘 것

이미 파이프라인 뼈대는 있다.

텔레그램 API (telegram_api_collect.py) → 인덱스 CSV / JSONL → OCR (run_ocr_sample.py) → 신호판 (신호판_2026-06-01.md 등) → (계획) 일일브리핑 → (계획) 1_브리핑/ 현재판 — 아직 폴더 없음

🌙 자동 (밤새 / 새벽)

  1. 텔레그램 새 메시지 수집
  2. 새 사진 OCR
  3. 신호판 MD 생성
  4. 요약 JSON 생성 → 대시보드 반영

🌅 당신이 하는 일 (아침 5분)

  1. Vercel 대시보드 열기
  2. 오늘 신호판 확인 (SM-S3 위주)
  3. 포트폴리오 현황 확인
  4. 오늘 매매 회의 카드 읽기
  5. "실행" 또는 "관망"만 결정

📅 가끔 (주 1회)

  1. 포트폴리오 보유 현황 MD 업데이트
  2. 칼럼에서 관련 글 1편 읽기
  3. 자산네제곱 단계별 학습 진행

🚀 단계별 구현 제안

1단계 MVP (1~2일, 바로 효과)

  • dashboard/ 폴더에 Next.js 프로젝트 생성
  • 기존 신호판_*.md, CSV를 읽어 정적 JSON 생성 스크립트
  • Vercel 배포 + 간단 비밀번호 (middleware 또는 Vercel Password Protection)
  • 수동: Python 스크립트 실행 → JSON 커밋 → 대시보드 자동 갱신

→ 지금 당장 "매일 볼 창" 만들 수 있음

2단계 반자동 (약 1주)

  • Windows 작업 스케줄러: 매일 06:00 수집 스크립트 실행
  • build_dashboard.py가 MD → JSON 변환
  • GitHub push → Vercel 자동 재배포
  • 1_브리핑/ 폴더 생성 (문서에만 있고 실제 폴더는 아직 없음)

3단계 준완전 자동 (2~3주)

  • GitHub Actions cron (또는 Railway 소형 worker)
  • OCR·신호 태그화 자동화 확대
  • 포트폴리오 입력 폼 (대시보드에서 MD 업데이트)
  • "오늘 매매할 것은?" 버튼 → Cursor/API로 토론 브리핑 생성

4단계 풀 자동 (선택)

  • Supabase에 일별 스냅샷 저장
  • 텔레그램 알림: "오늘 SM-S3 신호 3개"
  • 차트·히스토리 비교
